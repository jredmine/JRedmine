# 附件上传路径配置说明

## 问题描述

在 Tomcat 等容器环境中，如果使用相对路径配置附件存储路径，可能会遇到以下错误：

```
java.io.FileNotFoundException: .../tomcat.8088.../work/Tomcat/localhost/ROOT/files/2026/01/xxx (No such file or directory)
```

这是因为相对路径在容器环境中会被解析到容器的工作目录，而不是项目目录。

## 解决方案

### 方案1：使用绝对路径（推荐）

在配置文件中直接指定绝对路径：

```yaml
attachment:
  storage:
    path: /data/jredmine/files  # Linux/Mac 绝对路径
    # 或
    path: D:\jredmine\files     # Windows 绝对路径
```

**优点：**
- 路径明确，不会受运行环境影响
- 适合生产环境

### 方案2：使用相对路径 + 系统属性

如果使用相对路径，系统会自动基于 `user.home` 目录解析：

```yaml
attachment:
  storage:
    path: files  # 相对路径，会解析为 ~/files
```

可以通过 JVM 系统属性指定基础路径：

```bash
# 启动时指定
java -Dattachment.storage.base=/data/jredmine -jar app.jar

# 或在 application.yml 中配置
spring:
  application:
    name: JRedmine
  # 可以通过环境变量或系统属性覆盖
```

### 方案3：使用项目根目录

如果需要基于项目根目录，可以在启动时设置系统属性：

```bash
java -Dattachment.storage.base=/path/to/project -jar app.jar
```

## 配置示例

### 开发环境（application-local.yml）

```yaml
attachment:
  storage:
    path: files  # 相对路径，会解析为 ~/files
  max-file-size: 10485760
  allowed-extensions:
```

### 生产环境（application-prod.yml）

```yaml
attachment:
  storage:
    path: /data/jredmine/files  # 绝对路径
  max-file-size: 10485760
  allowed-extensions: jpg,jpeg,png,gif,pdf,doc,docx,xls,xlsx,txt,zip,rar
```

## 路径解析逻辑

代码中的路径解析逻辑：

```java
private Path getStoragePath() {
    Path path = Paths.get(storagePath);
    // 如果是相对路径，则基于用户目录或项目目录
    if (!path.isAbsolute()) {
        // 优先使用系统属性指定的路径
        String basePath = System.getProperty("attachment.storage.base", 
                System.getProperty("user.home", "."));
        path = Paths.get(basePath, storagePath);
    }
    return path;
}
```

**解析规则：**
1. 如果配置的是绝对路径，直接使用
2. 如果是相对路径：
   - 优先使用系统属性 `attachment.storage.base`
   - 如果没有，使用 `user.home`（用户主目录）
   - 最终路径 = basePath + storagePath

## 目录权限

确保存储目录有正确的读写权限：

```bash
# Linux/Mac
mkdir -p /data/jredmine/files
chmod 755 /data/jredmine/files
chown -R appuser:appuser /data/jredmine/files

# 或者使用用户目录
mkdir -p ~/files
chmod 755 ~/files
```

## 验证配置

启动应用后，查看日志确认路径：

```
文件保存成功: /data/jredmine/files/2026/01/xxx
```

如果看到路径正确，说明配置成功。

## 常见问题

### Q1: 为什么使用相对路径会失败？

A: 在 Tomcat 等容器环境中，相对路径会基于容器的工作目录解析，而不是项目目录。工作目录通常是临时目录，重启后可能变化。

### Q2: 如何查看当前使用的路径？

A: 查看应用日志，文件保存时会输出完整路径。或者添加日志：

```java
log.info("附件存储路径: {}", getStoragePath().toAbsolutePath());
```

### Q3: 生产环境推荐使用什么路径？

A: 推荐使用绝对路径，指向固定的数据目录，如 `/data/jredmine/files` 或 `/var/lib/jredmine/files`。

### Q4: 如何迁移已有文件？

A: 如果更改了存储路径，需要：
1. 停止应用
2. 将旧目录的文件复制到新目录
3. 更新配置
4. 重启应用

## 总结

- ✅ **推荐使用绝对路径**，避免路径解析问题
- ✅ **确保目录存在且有读写权限**
- ✅ **生产环境使用固定数据目录**
- ✅ **开发环境可以使用相对路径或用户目录**
