## 📊 数据库表结构分析

### 1. projects 表（项目主表）

```sql
CREATE TABLE `projects` (
  `id` int NOT NULL AUTO_INCREMENT,
  `name` varchar(255) NOT NULL DEFAULT '',              -- 项目名称
  `description` text,                                    -- 项目描述
  `homepage` varchar(255) DEFAULT '',                    -- 项目主页
  `is_public` tinyint(1) NOT NULL DEFAULT '1',          -- 是否公开（1=公开，0=私有）
  `parent_id` int DEFAULT NULL,                         -- 父项目ID（支持子项目）
  `created_on` timestamp NULL DEFAULT NULL,             -- 创建时间
  `updated_on` timestamp NULL DEFAULT NULL,             -- 更新时间
  `identifier` varchar(255) DEFAULT NULL,                -- 项目标识符（URL友好）
  `status` int NOT NULL DEFAULT '1',                     -- 项目状态（1=活跃，5=关闭，9=归档）
  `lft` int DEFAULT NULL,                                -- 左值（树形结构）
  `rgt` int DEFAULT NULL,                                 -- 右值（树形结构）
  `inherit_members` tinyint(1) NOT NULL DEFAULT '0',     -- 是否继承父项目成员
  `default_version_id` int DEFAULT NULL,                 -- 默认版本ID
  `default_assigned_to_id` int DEFAULT NULL,            -- 默认分配给用户ID
  `default_issue_query_id` int DEFAULT NULL,             -- 默认任务查询ID
  PRIMARY KEY (`id`),
  KEY `index_projects_on_lft` (`lft`),
  KEY `index_projects_on_rgt` (`rgt`)
)
```

**关键字段说明：**
- `parent_id`: 支持子项目，形成项目树
- `lft`, `rgt`: 嵌套集合模型（Nested Set Model），用于高效查询树形结构
- `status`: 项目状态（1=活跃，5=关闭，9=归档）
- `identifier`: 项目标识符，用于URL（如：`my-project`）
- `inherit_members`: 是否继承父项目的成员

### 2. members 表（项目成员表）

```sql
CREATE TABLE `members` (
  `id` int NOT NULL AUTO_INCREMENT,
  `user_id` int NOT NULL DEFAULT '0',                   -- 用户ID
  `project_id` int NOT NULL DEFAULT '0',                 -- 项目ID
  `created_on` timestamp NULL DEFAULT NULL,              -- 加入时间
  `mail_notification` tinyint(1) NOT NULL DEFAULT '0',  -- 邮件通知设置
  PRIMARY KEY (`id`),
  UNIQUE KEY `index_members_on_user_id_and_project_id` (`user_id`,`project_id`)
)
```

**说明：**
- 一个用户在一个项目中只能有一条记录
- 成员的角色通过 `member_roles` 表关联（见角色权限模块）

### 3. enabled_modules 表（项目启用的模块）

```sql
CREATE TABLE `enabled_modules` (
  `id` int NOT NULL AUTO_INCREMENT,
  `project_id` int DEFAULT NULL,                         -- 项目ID（NULL表示全局）
  `name` varchar(255) NOT NULL,                          -- 模块名称
  PRIMARY KEY (`id`),
  KEY `enabled_modules_project_id` (`project_id`)
)
```

**说明：**
- 控制项目启用哪些功能模块（如：issues、wiki、boards、documents等）
- `project_id` 为 NULL 表示全局模块

### 4. projects_trackers 表（项目跟踪器关联）

```sql
CREATE TABLE `projects_trackers` (
  `project_id` int NOT NULL DEFAULT '0',                -- 项目ID
  `tracker_id` int NOT NULL DEFAULT '0',                 -- 跟踪器ID
  UNIQUE KEY `projects_trackers_unique` (`project_id`,`tracker_id`)
)
```

**说明：**
- 关联项目使用的跟踪器（如：Bug、Feature、Support等）
- 一个项目可以使用多个跟踪器

## 🔴 核心接口（P0 - 必须实现）

### 1. 获取项目列表 ⚠️

**接口**: `GET /api/projects`

**状态**: ❌ 未实现

**功能**: 分页查询项目列表，支持多条件筛选

**查询参数**:
- `current` (Integer, 默认1) - 当前页码
- `size` (Integer, 默认10) - 每页数量
- `name` (String, 可选) - 项目名称（模糊查询）
- `status` (Integer, 可选) - 项目状态（1=活跃，5=关闭，9=归档）
- `isPublic` (Boolean, 可选) - 是否公开
- `parentId` (Long, 可选) - 父项目ID（查询子项目）

**响应**: `ApiResponse<PageResponse<ProjectListItemResponseDTO>>`

**权限**: 需要认证
- 公开项目：所有用户可见
- 私有项目：仅项目成员可见
- 系统管理员：可见所有项目

**业务逻辑**:
- 根据用户权限过滤项目
- 支持按名称模糊查询
- 支持按状态筛选
- 支持查询子项目

## 📌 注意事项

1. **项目标识符唯一性**: `identifier` 字段必须全局唯一，用于URL
2. **树形结构维护**: 添加/删除/移动子项目时需要更新 `lft` 和 `rgt` 值
3. **成员继承**: 如果 `inherit_members = true`，子项目会继承父项目的成员
4. **权限控制**: 所有接口都需要进行权限验证
5. **软删除**: 删除项目时使用归档（status = 9），而不是物理删除
6. **依赖关系**: 成员角色分配功能依赖角色权限模块
7. **项目可见性**: 公开项目所有用户可见，私有项目仅成员可见
