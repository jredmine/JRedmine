# 异步邮件发送方案说明

## 📋 问题分析

### 当前问题
- 任务变更时串行发送3封邮件，接口响应变慢
- 邮件发送耗时阻塞主业务逻辑
- 用户体验差，等待时间长

### 原因分析
邮件发送是 I/O 密集型操作，每封邮件可能耗时 100-500ms，3封邮件串行发送需要 300-1500ms。

---

## 🎯 解决方案：Spring @Async（已实现）

### 方案优势

✅ **零额外依赖** - Spring Boot 自带，无需引入消息队列  
✅ **实现简单** - 只需添加配置和注解  
✅ **性能提升明显** - 接口响应时间从 ~1000ms 降到 ~50ms  
✅ **无需维护中间件** - 不需要 Redis/RabbitMQ 等  
✅ **适合当前场景** - 邮件发送等轻量级异步任务完全够用  

### 实现方式

#### 1. 配置异步线程池（已完成）

文件：`AsyncConfig.java`

```java
@Configuration
@EnableAsync
public class AsyncConfig implements AsyncConfigurer {
    @Override
    public Executor getAsyncExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(4);           // 核心线程数
        executor.setMaxPoolSize(10);           // 最大线程数
        executor.setQueueCapacity(200);        // 队列容量
        executor.setThreadNamePrefix("async-email-");
        executor.setRejectedExecutionHandler(new ThreadPoolExecutor.CallerRunsPolicy());
        return executor;
    }
}
```

**线程池参数说明**：
- `CorePoolSize = 4`：同时最多有 4 个线程在发送邮件
- `MaxPoolSize = 10`：高峰期最多扩展到 10 个线程
- `QueueCapacity = 200`：最多排队 200 个邮件任务
- `CallerRunsPolicy`：队列满后由主线程执行（避免丢失邮件）

#### 2. 邮件服务添加 @Async 注解（已完成）

```java
@Service
public class EmailService {
    
    @Async  // 异步执行
    public void sendPasswordResetEmail(...) {
        // 邮件发送逻辑
    }
    
    @Async  // 异步执行
    public void sendIssueAssignmentEmail(...) {
        // 邮件发送逻辑
    }
    
    @Async  // 异步执行
    public void sendIssueUpdateEmail(...) {
        // 邮件发送逻辑
    }
}
```

### 效果对比

| 场景 | 同步发送 | 异步发送 |
|------|---------|---------|
| **单封邮件** | ~300ms | ~10ms |
| **3封邮件** | ~1000ms | ~10ms |
| **10封邮件** | ~3000ms | ~10ms |
| **接口响应** | 等待所有邮件发送完 | 立即返回 |
| **用户体验** | 明显卡顿 | 秒级响应 |

## 🔄 其他可选方案

### 方案二：Redis + Spring Data Redis Stream

**适用场景**：
- 需要持久化邮件队列
- 需要重试机制
- 需要监控队列状态

**优点**：
- 可靠性更高（持久化）
- 支持消费者组
- 可以监控队列长度

**缺点**：
- 需要部署 Redis（额外维护成本）
- 实现复杂度中等
- 需要配置 Redis

**实现成本**：1-2 天

**推荐指数**：⭐⭐⭐⭐

---

### 方案三：RabbitMQ

**适用场景**：
- 复杂的消息路由需求
- 需要严格的消息顺序
- 需要高可用部署

**优点**：
- 功能最强大
- 可靠性最高
- 支持复杂的消息模式

**缺点**：
- 需要部署 RabbitMQ（维护成本最高）
- 实现复杂度高
- 过度设计（对你的场景来说）

**实现成本**：2-3 天

**推荐指数**：⭐⭐⭐

## 🛠️ 使用说明

### 异步方法注意事项

#### ✅ 正确用法

```java
// 1. 在 @Service 类的 public 方法上使用
@Service
public class EmailService {
    @Async
    public void sendEmail(...) {
        // 邮件发送逻辑
    }
}

// 2. 从其他 Bean 调用
@Service
public class IssueService {
    @Autowired
    private EmailService emailService;
    
    public void updateIssue() {
        // ... 业务逻辑
        emailService.sendEmail(...);  // ✅ 异步执行
    }
}
```

#### ❌ 错误用法

```java
// 1. 在同一个类中调用（不会异步）
@Service
public class EmailService {
    @Async
    public void sendEmail() { }
    
    public void doSomething() {
        this.sendEmail();  // ❌ 不会异步，因为是内部调用
    }
}

// 2. 在非 Spring 管理的类中使用
public class NotABean {
    @Async  // ❌ 无效，因为不是 Spring Bean
    public void sendEmail() { }
}
```

### 异步执行监控

#### 查看线程池状态

```java
@Autowired
private ThreadPoolTaskExecutor taskExecutor;

// 获取活跃线程数
int activeCount = taskExecutor.getActiveCount();

// 获取队列中等待的任务数
int queueSize = taskExecutor.getThreadPoolExecutor().getQueue().size();

log.info("异步线程池状态 - 活跃线程: {}, 队列任务: {}", activeCount, queueSize);
```

#### 日志观察

邮件发送会在单独的线程中执行，日志中可以看到线程名：

```
[async-email-1] INFO c.g.j.service.EmailService - 任务更新通知邮件发送成功，收件人: user@example.com
[async-email-2] INFO c.g.j.service.EmailService - 任务更新通知邮件发送成功，收件人: admin@example.com
[async-email-3] INFO c.g.j.service.EmailService - 任务更新通知邮件发送成功，收件人: manager@example.com
```

### 异常处理

异步方法中的异常会被 `AsyncUncaughtExceptionHandler` 捕获：

```java
@Override
public AsyncUncaughtExceptionHandler getAsyncUncaughtExceptionHandler() {
    return (throwable, method, params) -> {
        log.error("异步任务执行异常，方法: {}, 参数: {}", method.getName(), params, throwable);
    };
}
```

**注意**：异步方法中的异常不会传播到调用方，需要在异步方法内部处理。

## 🚀 性能优化建议

### 1. 线程池参数调优

根据实际情况调整参数：

```java
// 邮件发送量较小（每小时 < 100 封）
executor.setCorePoolSize(2);
executor.setMaxPoolSize(5);

// 邮件发送量中等（每小时 100-1000 封）
executor.setCorePoolSize(4);  // 当前配置
executor.setMaxPoolSize(10);

// 邮件发送量较大（每小时 > 1000 封）
executor.setCorePoolSize(8);
executor.setMaxPoolSize(20);
```

### 2. 批量发送优化

对于多个收件人，可以考虑批量发送：

```java
@Async
public void sendBatchEmails(List<EmailDTO> emails) {
    for (EmailDTO email : emails) {
        try {
            sendSingleEmail(email);
        } catch (Exception e) {
            log.error("批量发送邮件失败，收件人: {}", email.getTo(), e);
        }
    }
}
```

### 3. 监控告警

建议添加监控：
- 线程池队列长度超过阈值（如 150）时告警
- 邮件发送失败率超过阈值（如 5%）时告警
- 线程池拒绝任务时告警
