# 任务关联关系类型说明与使用场景

## 概述

任务关联关系（Issue Relations）用于定义任务之间的各种关系，目前系统支持8种关联类型，但大部分关联类型的业务逻辑还未实现。

## 当前状态

### ✅ 已实现的功能

1. **基础功能**
   - ✅ 创建任务关联（`POST /api/issues/{id}/relations`）
   - ✅ 删除任务关联（`DELETE /api/issues/{id}/relations/{relationId}`）
   - ✅ 甘特图中使用关联关系展示依赖

2. **已使用的关联类型**
   - ✅ `precedes` - 前置（用于甘特图）
   - ✅ `follows` - 后置（用于甘特图）
   - ✅ `blocks` - 阻塞（甘特图中展示）
   - ✅ `blocked` - 被阻塞（甘特图中展示）

### ❌ 未完整实现的功能

1. **查询功能缺失**
   - ❌ 获取任务的所有关联关系列表
   - ❌ 任务详情中不返回关联关系
   - ❌ 批量查询关联关系

2. **业务逻辑未实现**
   - ❌ `duplicates/duplicated` - 重复关系未实现自动反向关联
   - ❌ `blocks/blocked` - 阻塞关系未实现状态限制
   - ❌ `copied_to/copied_from` - 复制关系未实现自动创建
   - ❌ 关联关系对任务状态/工作流的影响

3. **关联类型的特殊处理**
   - ❌ 对称关系（duplicates/duplicated）未自动创建反向关联
   - ❌ 阻塞关系（blocks/blocked）未限制任务操作
   - ❌ 前置关系（precedes/follows）未实现日期自动计算

## 关联类型详解

### 1. relates - 相关

**用途**: 表示两个任务相关，但没有明确的依赖或阻塞关系。

**特点**:
- 最简单的关联类型
- 不影响任务状态和工作流
- 仅用于信息展示

**应用场景**:
- 两个功能模块的任务有关联但不互相依赖
- 需要一起查看的相关任务
- 参考任务

**当前实现**: ✅ 基础创建/删除
**待完善**: 
- [ ] 在任务详情中展示相关任务列表
- [ ] 支持快速跳转到相关任务

---

### 2. duplicates / duplicated - 重复

**用途**: 表示任务重复，通常一个是重复项，需要关闭。

**特点**:
- **对称关系**: A duplicates B，则 B duplicated A
- 应该自动创建反向关联
- 通常需要关闭重复的任务

**应用场景**:
- 用户重复提交了相同的Bug
- 两个任务实际是同一个需求
- 合并重复任务

**当前实现**: ⚠️ 仅支持创建，未实现自动反向关联
**待完善**:
- [ ] 创建 A duplicates B 时，自动创建 B duplicated A
- [ ] 删除关联时，同时删除反向关联
- [ ] 在任务详情中显示"重复任务"和"被重复"的任务
- [ ] 重复任务应该提示关闭
- [ ] 支持合并重复任务功能

**实现建议**:
```java
// 创建重复关系时
if ("duplicates".equals(relationType)) {
    // 创建正向关系: A duplicates B
    createRelation(issueA, issueB, "duplicates");
    // 自动创建反向关系: B duplicated A
    createRelation(issueB, issueA, "duplicated");
}
```

### 3. blocks / blocked - 阻塞

**用途**: 表示一个任务阻塞了另一个任务，被阻塞的任务无法完成。

**特点**:
- **对称关系**: A blocks B，则 B blocked A
- 应该限制被阻塞任务的状态变更
- 阻塞任务解除前，被阻塞任务不应该能关闭

**应用场景**:
- Bug阻塞了功能开发
- 前置条件未满足
- 依赖的第三方服务未就绪

**当前实现**: ⚠️ 仅用于甘特图展示，未实现业务逻辑
**待完善**:
- [ ] 创建 A blocks B 时，自动创建 B blocked A
- [ ] 删除关联时，同时删除反向关联
- [ ] 被阻塞的任务（B）不能关闭，除非先解除阻塞
- [ ] 在任务详情中显示"阻塞"和"被阻塞"的任务列表
- [ ] 显示阻塞警告："该任务被 X 个任务阻塞"
- [ ] 尝试关闭被阻塞任务时提示错误

**实现建议**:
```java
// 更新任务状态时检查
public void updateIssueStatus(Long issueId, Integer newStatusId) {
    // 检查任务是否被阻塞
    List<IssueRelation> blockedRelations = issueRelationMapper.selectList(
        new LambdaQueryWrapper<IssueRelation>()
            .eq(IssueRelation::getIssueToId, issueId)
            .eq(IssueRelation::getRelationType, "blocked"));
    
    if (!blockedRelations.isEmpty() && isClosedStatus(newStatusId)) {
        // 检查阻塞任务是否都已关闭
        boolean hasOpenBlockers = blockedRelations.stream()
            .anyMatch(rel -> !isIssueClosed(rel.getIssueFromId()));
        
        if (hasOpenBlockers) {
            throw new BusinessException("该任务被其他任务阻塞，无法关闭");
        }
    }
}
```

### 4. precedes / follows - 前置/后置

**用途**: 表示任务的先后顺序，主要用于甘特图和项目排期。

**特点**:
- **对称关系**: A precedes B，则 B follows A
- 支持延迟天数（delay）
- 影响甘特图的任务排列
- 可以用于自动计算任务日期

**应用场景**:
- 项目排期和进度管理
- 任务依赖链
- 自动计算开始/结束日期

**当前实现**: ✅ 已用于甘特图展示依赖关系
**待完善**:
- [ ] 创建 A precedes B 时，自动创建 B follows A
- [ ] 删除关联时，同时删除反向关联
- [ ] 根据前置任务的截止日期和delay，自动计算后续任务的开始日期
- [ ] 前置任务延期时，自动提醒后续任务也需要调整
- [ ] 在任务详情中显示前置/后置任务列表
- [ ] 支持拖拽甘特图时自动更新关联任务日期

**实现建议**:
```java
// 自动计算日期
public void calculateDatesBasedOnPrecedence(Long issueId) {
    // 查找所有前置任务
    List<IssueRelation> precedingRelations = issueRelationMapper.selectList(
        new LambdaQueryWrapper<IssueRelation>()
            .eq(IssueRelation::getIssueToId, issueId)
            .eq(IssueRelation::getRelationType, "follows"));
    
    if (!precedingRelations.isEmpty()) {
        // 找到最晚的前置任务截止日期
        LocalDate latestDueDate = precedingRelations.stream()
            .map(rel -> {
                Issue precedingIssue = issueMapper.selectById(rel.getIssueFromId());
                LocalDate dueDate = precedingIssue.getDueDate();
                Integer delay = rel.getDelay() != null ? rel.getDelay() : 0;
                return dueDate != null ? dueDate.plusDays(delay) : null;
            })
            .filter(Objects::nonNull)
            .max(LocalDate::compareTo)
            .orElse(null);
        
        if (latestDueDate != null) {
            // 更新当前任务的开始日期
            Issue issue = issueMapper.selectById(issueId);
            if (issue.getStartDate() == null || issue.getStartDate().isBefore(latestDueDate)) {
                issue.setStartDate(latestDueDate);
                issueMapper.updateById(issue);
            }
        }
    }
}
```

### 5. copied_to / copied_from - 复制关系

**用途**: 记录任务的复制关系，追踪任务来源。

**特点**:
- **对称关系**: A copied_to B，则 B copied_from A
- 应该在复制任务时自动创建
- 用于追溯和审计

**应用场景**:
- 复制任务到其他项目
- 基于已有任务创建新任务
- 追踪任务来源

**当前实现**: ❌ 完全未实现
**待完善**:
- [ ] 复制任务时自动创建 copied_to 和 copied_from 关联
- [ ] 在任务详情中显示"复制自"和"已复制到"
- [ ] 支持快速查看原始任务和所有副本
- [ ] 原始任务更新时，可选择同步更新副本

**实现建议**:
```java
// 在 copyIssue 方法中
public IssueDetailResponseDTO copyIssue(Long sourceId, IssueCopyRequestDTO requestDTO) {
    // ... 复制任务逻辑 ...
    Issue newIssue = createCopiedIssue(sourceIssue, requestDTO);
    
    // 创建复制关系
    IssueRelation copiedToRelation = new IssueRelation();
    copiedToRelation.setIssueFromId(sourceId.intValue());
    copiedToRelation.setIssueToId(newIssue.getId().intValue());
    copiedToRelation.setRelationType("copied_to");
    issueRelationMapper.insert(copiedToRelation);
    
    // 自动创建反向关系
    IssueRelation copiedFromRelation = new IssueRelation();
    copiedFromRelation.setIssueFromId(newIssue.getId().intValue());
    copiedFromRelation.setIssueToId(sourceId.intValue());
    copiedFromRelation.setRelationType("copied_from");
    issueRelationMapper.insert(copiedFromRelation);
    
    return convertToDTO(newIssue);
}
```
