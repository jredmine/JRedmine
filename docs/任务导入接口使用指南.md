# 任务导入接口使用指南

## 概述

任务导入功能允许用户通过 Excel 文件批量创建任务。系统提供两种导入方式：

1. **文件上传方式**：通过 HTTP multipart/form-data 上传 Excel 文件
2. **文件路径方式**：直接指定服务器上的 Excel 文件路径

## 接口说明

### 1. 下载导入模板

**接口地址**：`GET /api/issues/import-template`

**描述**：下载任务导入的 Excel 模板文件，模板包含所有支持导入的字段和示例数据。

**请求示例**：

```bash
curl -X GET 'http://localhost:8088/api/issues/import-template' \
  -H 'Authorization: Bearer {token}' \
  -o issues_import_template.xlsx
```

**模板字段说明**：

| 列名 | 是否必填 | 说明 | 示例 |
|------|---------|------|------|
| 任务标题* | 是 | 任务的标题 | 修复登录页面Bug |
| 跟踪器 | 否 | 任务类型（Bug、Feature、Support等），留空默认为Bug | Bug |
| 状态 | 否 | 任务状态（New、In Progress等），留空默认为New | New |
| 优先级 | 否 | 任务优先级（Low、Normal、High等），留空默认为Normal | High |
| 指派人(登录名) | 否 | 被指派人的登录名 | admin |
| 描述 | 否 | 任务的详细描述 | 登录页面无法正常显示 |
| 开始日期(yyyy-MM-dd) | 否 | 任务开始日期，格式：yyyy-MM-dd | 2026-01-10 |
| 截止日期(yyyy-MM-dd) | 否 | 任务截止日期，格式：yyyy-MM-dd | 2026-01-15 |
| 预估工时 | 否 | 预估完成任务所需工时（小时） | 4 |
| 完成度(0-100) | 否 | 任务完成百分比，0-100之间，留空默认为0 | 0 |

### 2. 导入任务（文件上传方式）

**接口地址**：`POST /api/issues/import`

**描述**：通过上传 Excel 文件批量导入任务到指定项目。

**请求参数**：
- `projectId`（必填）：项目ID
- `file`（必填）：Excel 文件（.xlsx 或 .xls 格式）

**请求示例**：

```bash
curl -X POST 'http://localhost:8088/api/issues/import?projectId=1' \
  -H 'Authorization: Bearer {token}' \
  -H 'Content-Type: multipart/form-data' \
  -F 'file=@/Users/username/Downloads/issues_to_import.xlsx'
```

**注意**：
- Swagger UI 目前不支持文件上传测试，请使用 curl、Postman 或其他 HTTP 客户端
- 文件大小建议不超过 10MB
- 单次导入建议不超过 1000 行数据

### 3. 导入任务（文件路径方式）

**接口地址**：`POST /api/issues/import-from-path`

**描述**：通过指定服务器上的 Excel 文件路径批量导入任务。此方式适用于服务器本地文件导入场景。

**请求参数**：
- `projectId`（必填）：项目ID
- `filePath`（必填）：服务器上 Excel 文件的绝对路径

**请求示例**：

```bash
curl -X POST 'http://localhost:8088/api/issues/import-from-path' \
  -H 'Authorization: Bearer {token}' \
  -H 'Content-Type: application/json' \
  -d '{
    "projectId": 1,
    "filePath": "/Users/panfeng/Downloads/issues_import_template.xlsx"
  }'
```

**cURL 命令示例（使用查询参数）**：

```bash
curl -X POST \
  'http://localhost:8088/api/issues/import-from-path?projectId=1&filePath=/Users/panfeng/Downloads/issues_import_template.xlsx' \
  -H 'accept: */*' \
  -H 'Authorization: Bearer eyJhbGci...' \
  -H 'Content-Type: application/json'
```

```json
{
  "code": 200,

  "data": {
```

| 字段 | 类型 | 说明 |
|------|------|------|
| total | Integer | Excel 文件中的总行数（不含标题行） |

| errors[].message | String | 错误信息 |

## 导入规则

### 1. 数据验证

- **必填字段**：任务标题不能为空
- **跟踪器**：必须存在于系统中，不存在则导入失败
- **状态**：必须存在于系统中，不存在则导入失败
- **优先级**：必须存在于系统中，不存在则导入失败
- **指派人**：必须是系统中存在的用户登录名，不存在则导入失败
- **日期格式**：必须为 `yyyy-MM-dd` 格式，如 `2026-01-10`
- **预估工时**：必须是数字，可以包含小数
- **完成度**：必须是 0-100 之间的整数

### 2. 默认值

- **跟踪器**：默认为 `Bug` (ID=1)
- **状态**：默认为 `New` (ID=1)
- **优先级**：默认为 `Normal` (ID=2)
- **完成度**：默认为 `0`

### 3. 权限要求

- 需要认证（Bearer Token）
- 需要对目标项目拥有 `add_issues` 权限
- 系统管理员可导入任何项目的任务

### 4. 错误处理

- **部分成功**：某些行导入失败不影响其他行的导入
- **错误详情**：所有失败的行都会在响应中返回详细的错误信息
- **事务控制**：每个任务独立导入，互不影响

### 5. 自动创建日志

- 每个成功导入的任务都会自动创建一条日志记录
- 日志备注为："通过导入创建"
- 日志类型为任务创建类型

## 使用场景

### 场景 1：批量创建新项目任务

1. 下载模板文件
2. 填写任务信息（可以使用 Excel 编辑）
3. 上传文件或指定文件路径
4. 查看导入结果

### 场景 2：从其他系统迁移任务

1. 将其他系统的任务数据导出为 CSV
2. 转换为 Excel 格式并按模板格式调整
3. 使用文件路径方式批量导入
4. 检查错误信息并修正

### 场景 3：定期批量导入

1. 系统管理员在服务器上准备好 Excel 文件
2. 使用文件路径方式导入
3. 通过 API 响应获取导入结果
4. 可以集成到自动化脚本中

## 常见问题

### Q1：为什么有两种导入方式？

**A**：
- **文件上传方式**：适用于通过浏览器或 HTTP 客户端上传文件的场景
- **文件路径方式**：适用于文件已在服务器上的场景，可以避免文件传输开销

### Q2：Swagger UI 中无法测试文件上传接口？

**A**：Swagger UI 暂不支持文件上传测试。建议使用：
- cURL 命令行工具
- Postman 或 Insomnia 等 API 测试工具
- 或使用文件路径方式（`/import-from-path` 接口）进行测试

### Q3：导入失败的行会影响其他行吗？

**A**：不会。每行任务独立处理，某行失败不影响其他行的导入。所有失败的行会在响应的 `errors` 字段中返回详细错误信息。

### Q4：可以导入子任务吗？

**A**：当前版本不支持在导入时直接指定父任务。需要先导入所有任务，然后通过更新任务接口设置父任务关系。

### Q5：导入的任务会发送邮件通知吗？

**A**：当前导入过程不会触发邮件通知，避免批量导入时发送大量邮件。

### Q6：支持哪些 Excel 格式？

**A**：支持以下格式：
- `.xlsx`（Excel 2007+，推荐）
- `.xls`（Excel 97-2003）

### Q7：导入性能如何？

**A**：
- 单次导入建议不超过 1000 行
- 每行处理时间约 50-200ms（取决于数据复杂度）
- 1000 行数据预计需要 1-3 分钟

## 技术实现

### 依赖库

```xml
<dependency>
    <groupId>org.apache.poi</groupId>
    <artifactId>poi</artifactId>
    <version>5.2.3</version>
</dependency>
<dependency>
    <groupId>org.apache.poi</groupId>
    <artifactId>poi-ooxml</artifactId>
    <version>5.2.3</version>
</dependency>
```

### 核心方法

- `IssueService.importIssuesFromExcel(Long, MultipartFile)`：文件上传导入
- `IssueService.importIssuesFromPath(Long, String)`：文件路径导入
- `IssueService.generateImportTemplate()`：生成导入模板

### 数据处理流程

1. 验证文件格式和项目权限
2. 读取 Excel 文件（跳过标题行）
3. 逐行解析数据
4. 验证必填字段和数据格式
5. 查找关联实体（跟踪器、状态、优先级、用户）
6. 创建任务并处理树形结构
7. 记录创建日志
8. 统计导入结果
