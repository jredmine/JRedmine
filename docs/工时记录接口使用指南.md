# 工时记录接口使用指南

## 概述

工时记录（Time Entry）模块用于记录团队成员在项目或任务上花费的工作时间。本文档介绍工时记录相关接口的使用方法。

## 接口列表

### 1. 创建工时记录

**接口地址：** `POST /api/time-entries`

**权限要求：** 管理员（`ROLE_ADMIN`）或拥有 `log_time` 权限

**请求参数：**

```json
{
  "projectId": 1,           // 必填：项目ID
  "issueId": 1,            // 可选：任务ID（如果不关联具体任务可不填）
  "userId": 2,             // 可选：工作人员ID（不填则为当前登录用户）
  "hours": 2.5,            // 必填：工时（小时），必须大于0
  "spentOn": "2026-01-12", // 必填：工作日期，格式：yyyy-MM-dd
  "activityId": 9,         // 必填：活动类型ID
  "comments": "完成了任务开发工作"  // 可选：备注说明，最长1024字符
}
```

**响应示例：**

```json
{
  "code": 200,
  "message": "工时记录创建成功",
  "data": {
    "id": 1,
    "project": {
      "id": 1,
      "name": "示例项目",
      "identifier": "demo-project"
    },
    "issueId": 1,
    "issueSubject": "实现用户登录功能",
    "user": {
      "id": 2,
      "login": "developer",
      "firstname": "张",
      "lastname": "三"
    },
    "author": {
      "id": 1,
      "login": "admin",
      "firstname": "管理",
      "lastname": "员"
    },
    "hours": 2.5,
    "spentOn": "2026-01-12",
    "activityId": 9,
    "activityName": "开发",
    "comments": "完成了任务开发工作",
    "createdOn": "2026-01-12 10:30:00",
    "updatedOn": "2026-01-12 10:30:00"
  },
  "success": true
}
```

**字段说明：**

- `projectId`：工时记录所属的项目
- `issueId`：关联的任务（可选）。如果填写，系统会验证任务是否存在且属于指定项目
- `userId`：实际工作人员。如果不填，默认为当前登录用户
- `hours`：工作时长（小时），可以是小数，如 2.5 表示 2 小时 30 分钟
- `spentOn`：工作日期，用于统计和报表
- `activityId`：活动类型，引用 `enumerations` 表中 `type='TimeEntryActivity'` 的记录
- `comments`：备注说明，用于描述具体工作内容
- `author`：记录创建者（系统自动设置为当前登录用户）

**注意事项：**

1. 系统会自动计算并保存年份（tyear）、月份（tmonth）、周数（tweek），用于后续统计查询
2. 如果指定了 `issueId`，系统会验证任务是否存在且属于指定的项目
3. 如果指定了 `userId`，系统会验证该用户是否存在
4. `activityId` 必须是有效的工时活动类型

### 3. 查询工时记录列表

**接口地址：** `GET /api/time-entries`

**权限要求：** 管理员（`ROLE_ADMIN`）或拥有 `view_time_entries` 权限

**查询参数：** （所有参数均为可选）

| 参数名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| projectId | Long | 项目ID | 1 |
| issueId | Long | 任务ID | 5 |
| userId | Long | 工作人员ID | 2 |
| activityId | Long | 活动类型ID | 9 |
| startDate | LocalDate | 开始日期（含） | 2026-01-01 |
| endDate | LocalDate | 结束日期（含） | 2026-01-31 |
| year | Integer | 年份 | 2026 |
| month | Integer | 月份（1-12） | 1 |
| pageNum | Integer | 当前页码（从1开始） | 1 |
| pageSize | Integer | 每页大小 | 20 |
| sortBy | String | 排序字段（spent_on, created_on, hours） | spent_on |
| sortOrder | String | 排序方向（asc, desc） | desc |

**响应示例：**

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "records": [
      {
        "id": 1,
        "project": {
          "id": 1,
          "name": "示例项目",
          "identifier": "demo-project"
        },
        "issueId": 1,
        "issueSubject": "实现用户登录功能",
        "user": {
          "id": 2,
          "login": "developer",
          "firstname": "张",
          "lastname": "三"
        },
        "author": {
          "id": 1,
          "login": "admin",
          "firstname": "管理",
          "lastname": "员"
        },
        "hours": 2.5,
        "spentOn": "2026-01-12",
        "activityId": 9,
        "activityName": "开发",
        "comments": "完成了任务开发工作",
        "createdOn": "2026-01-12 10:30:00",
        "updatedOn": "2026-01-12 10:30:00"
      }
    ],
    "total": 100,
    "current": 1,
    "size": 20,
    "pages": 5
  },
  "success": true
}
```

**查询示例：**

```bash
# 查询所有工时记录（分页）
curl -X GET "http://localhost:8088/api/time-entries?pageNum=1&pageSize=20" \
  -H "Authorization: Bearer YOUR_TOKEN"

# 查询指定项目的工时记录
curl -X GET "http://localhost:8088/api/time-entries?projectId=1" \
  -H "Authorization: Bearer YOUR_TOKEN"

# 查询指定任务的工时记录
curl -X GET "http://localhost:8088/api/time-entries?issueId=5" \
  -H "Authorization: Bearer YOUR_TOKEN"

# 查询指定用户的工时记录
curl -X GET "http://localhost:8088/api/time-entries?userId=2" \
  -H "Authorization: Bearer YOUR_TOKEN"

# 按日期范围查询
curl -X GET "http://localhost:8088/api/time-entries?startDate=2026-01-01&endDate=2026-01-31" \
  -H "Authorization: Bearer YOUR_TOKEN"

# 查询指定年月的工时记录
curl -X GET "http://localhost:8088/api/time-entries?year=2026&month=1" \
  -H "Authorization: Bearer YOUR_TOKEN"

# 组合条件查询并排序
curl -X GET "http://localhost:8088/api/time-entries?projectId=1&year=2026&month=1&sortBy=hours&sortOrder=desc" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 4. 更新工时记录

**接口地址：** `PUT /api/time-entries/{id}`

**权限要求：** 管理员（`ROLE_ADMIN`）或拥有 `edit_time_entries` 权限

**路径参数：**

- `id`：工时记录ID

**请求参数：** （所有字段均为可选，只需要传入需要修改的字段）

```json
{
  "projectId": 1,          // 可选：项目ID
  "issueId": 1,            // 可选：任务ID
  "userId": 2,             // 可选：工作人员ID
  "hours": 3.5,            // 可选：工时（小时），必须大于0
  "spentOn": "2026-01-12", // 可选：工作日期，格式：yyyy-MM-dd
  "activityId": 9,         // 可选：活动类型ID
  "comments": "更新后的备注" // 可选：备注说明，最长1024字符
}
```

**响应示例：**

```json
{
  "code": 200,
  "message": "工时记录更新成功",
  "data": {
    "id": 1,
    "project": {
      "id": 1,
      "name": "示例项目",
      "identifier": "demo-project"
    },
    "issueId": 1,
    "issueSubject": "实现用户登录功能",
    "user": {
      "id": 2,
      "login": "developer",
      "firstname": "张",
      "lastname": "三"
    },
    "author": {
      "id": 1,
      "login": "admin",
      "firstname": "管理",
      "lastname": "员"
    },
    "hours": 3.5,
    "spentOn": "2026-01-12",
    "activityId": 9,
    "activityName": "开发",
    "comments": "更新后的备注",
    "createdOn": "2026-01-12 10:30:00",
    "updatedOn": "2026-01-12 15:45:00"
  },
  "success": true
}
```

**注意事项：**

1. **权限控制**：只能更新自己创建的工时记录（`authorId` 等于当前用户），管理员可以更新任何记录
2. **部分更新**：所有字段都是可选的，只需要传入需要修改的字段
3. **自动重算**：如果更新了 `spentOn` 日期，系统会自动重新计算年、月、周字段
4. **验证规则**：与创建接口相同，会验证项目、任务、用户、活动类型的有效性

### 5. 删除工时记录

**接口地址：** `DELETE /api/time-entries/{id}`

**权限要求：** 管理员（`ROLE_ADMIN`）或拥有 `edit_time_entries` 权限

**路径参数：**

- `id`：工时记录ID

**响应示例：**

```json
{
  "code": 200,
  "message": "工时记录删除成功",
  "data": null,
  "success": true
}
```

**注意事项：**

1. **权限控制**：只能删除自己创建的工时记录（`authorId` 等于当前用户），管理员可以删除任何记录
2. **物理删除**：工时记录会从数据库中永久删除，无法恢复
3. **谨慎操作**：删除操作不可逆，建议在删除前进行确认

### 6. 工时汇总统计

**接口地址：** `GET /api/time-entries/summary`

**权限要求：** 管理员（`ROLE_ADMIN`）或拥有 `view_time_entries` 权限

**查询参数：** （所有参数均为可选）

```
projectId     // 项目ID
issueId       // 任务ID
userId        // 用户ID
activityId    // 活动类型ID
startDate     // 开始日期，格式：yyyy-MM-dd
endDate       // 结束日期，格式：yyyy-MM-dd
year          // 年份，如：2026
month         // 月份，1-12
```

**响应示例：**

```json
{
  "code": 200,
  "message": "工时汇总统计获取成功",
  "data": {
    "totalHours": 156.5,      // 总工时（小时）
    "totalCount": 42,          // 记录总数
    "averageHours": 3.73,      // 平均工时（小时）
    "minHours": 0.5,           // 最小工时（小时）
    "maxHours": 8.0            // 最大工时（小时）
  },
  "success": true
}
```

**应用场景：**

1. **项目总工时统计**：查询某个项目的总工时
2. **个人工时汇总**：查询某个用户的工时统计
3. **时间段统计**：查询某个时间段内的工时情况
4. **成本核算**：根据工时统计项目成本

### 7. 工时分组统计

**接口地址：** `GET /api/time-entries/statistics`

**权限要求：** 管理员（`ROLE_ADMIN`）或拥有 `view_time_entries` 权限

**查询参数：** （所有参数均为可选）

```
groupBy       // 分组维度：project-按项目, user-按用户, activity-按活动类型, date-按日期
projectId     // 项目ID
issueId       // 任务ID
userId        // 用户ID
activityId    // 活动类型ID
startDate     // 开始日期，格式：yyyy-MM-dd
endDate       // 结束日期，格式：yyyy-MM-dd
year          // 年份，如：2026
month         // 月份，1-12
```

**响应示例（按项目分组）：**

```json
{
  "code": 200,
  "message": "工时分组统计获取成功",
  "data": {
    "totalHours": 156.5,       // 总工时（小时）
    "totalCount": 42,           // 记录总数
    "averageHours": 3.73,       // 平均工时（小时）
    "groups": [
      {
        "groupKey": "1",         // 项目ID
        "groupName": "示例项目",  // 项目名称
        "hours": 98.5,           // 该项目工时
        "count": 28,             // 该项目记录数
        "percentage": 62.94      // 占总工时百分比
      },
      {
        "groupKey": "2",
        "groupName": "测试项目",
        "hours": 58.0,
        "count": 14,
        "percentage": 37.06
      }
    ]
  },
  "success": true
}
```

**分组维度说明：**

| 分组维度 | groupBy值 | 说明 | groupKey | groupName |
|---------|-----------|------|----------|-----------|
| 按项目 | project | 按项目统计工时分布 | 项目ID | 项目名称 |
| 按用户 | user | 按用户统计工时分布 | 用户ID | 用户登录名 |
| 按活动类型 | activity | 按活动类型统计工时分布 | 活动类型ID | 活动类型名称 |
| 按日期 | date | 按日期统计工时分布 | 日期 | 日期 |

**应用场景：**

1. **项目工时分布**：查看哪些项目占用工时最多
2. **人员工时分布**：查看团队成员的工时分配
3. **活动类型分布**：查看开发、测试、设计等活动的工时占比
4. **时间趋势分析**：按日期查看工时变化趋势
5. **组合分析**：在特定项目内按用户分组，查看团队贡献

## 活动类型说明

活动类型（Activity）用于分类不同类型的工作，常见的活动类型包括：

- **开发**：编写代码、实现功能
- **设计**：UI/UX 设计、架构设计
- **测试**：功能测试、性能测试
- **文档**：编写文档、更新说明
- **管理**：项目管理、团队协调
- **支持**：技术支持、问题排查

活动类型存储在 `enumerations` 表中，`type` 字段为 `TimeEntryActivity`。

## 使用场景

### 场景 1：记录任务开发工时

开发人员完成某个任务后，记录花费的工作时间：

```bash
curl -X POST "http://localhost:8088/api/time-entries" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "projectId": 1,
    "issueId": 5,
    "hours": 3.0,
    "spentOn": "2026-01-12",
    "activityId": 9,
    "comments": "完成用户注册功能，包括表单验证和邮箱确认"
  }'
```

### 场景 2：记录项目管理工时

项目经理记录项目管理工作时间（不关联具体任务）：

```bash
curl -X POST "http://localhost:8088/api/time-entries" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "projectId": 1,
    "hours": 1.5,
    "spentOn": "2026-01-12",
    "activityId": 10,
    "comments": "项目周会、进度跟踪"
  }'
```

### 场景 3：为团队成员记录工时

管理员或项目经理为其他成员记录工时：

```bash
curl -X POST "http://localhost:8088/api/time-entries" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "projectId": 1,
    "userId": 5,
    "issueId": 8,
    "hours": 2.0,
    "spentOn": "2026-01-11",
    "activityId": 9,
    "comments": "代理记录：完成前端界面开发"
  }'
```

### 场景 4：修改工时记录

修改之前记录的工时：

```bash
curl -X PUT "http://localhost:8088/api/time-entries/1" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "hours": 3.5,
    "comments": "实际花费时间更长，更新为3.5小时"
  }'
```

### 场景 5：更改工时所属任务

将工时记录关联到另一个任务：

```bash
curl -X PUT "http://localhost:8088/api/time-entries/1" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "issueId": 10,
    "comments": "这部分工作实际属于另一个任务"
  }'
```

### 场景 6：查看项目工时统计

查询某个项目的所有工时记录，用于统计分析：

```bash
curl -X GET "http://localhost:8088/api/time-entries?projectId=1&pageNum=1&pageSize=100" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 场景 7：查看个人本月工时

查询当前用户本月的工时记录：

```bash
curl -X GET "http://localhost:8088/api/time-entries?userId=2&year=2026&month=1" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 场景 8：导出工时报表

查询指定时间范围的工时记录，用于导出报表：

```bash
curl -X GET "http://localhost:8088/api/time-entries?startDate=2026-01-01&endDate=2026-01-31&pageNum=1&pageSize=1000&sortBy=spent_on&sortOrder=asc" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 场景 9：删除错误的工时记录

删除记录错误的工时：



生成用户的详细工时报表，用于绩效评估：

导出用户工时报表为PDF文件：

## 数据库结构

工时记录存储在 `time_entries` 表中：

```sql
CREATE TABLE `time_entries` (
  `id` int NOT NULL AUTO_INCREMENT,
  `project_id` int NOT NULL,
  `author_id` int DEFAULT NULL,          -- 记录创建者
  `user_id` int NOT NULL,                -- 实际工作人员
  `issue_id` int DEFAULT NULL,           -- 关联任务（可选）
  `hours` float NOT NULL,                -- 工时（小时）
  `comments` varchar(1024) DEFAULT NULL, -- 备注
  `activity_id` int NOT NULL,            -- 活动类型
  `spent_on` date NOT NULL,              -- 工作日期
  `tyear` int NOT NULL,                  -- 年份（冗余字段）
  `tmonth` int NOT NULL,                 -- 月份（冗余字段）
  `tweek` int NOT NULL,                  -- 周数（冗余字段）
  `created_on` datetime NOT NULL,
  `updated_on` datetime NOT NULL,
  PRIMARY KEY (`id`),
  KEY `time_entries_project_id` (`project_id`),
  KEY `time_entries_issue_id` (`issue_id`),
  KEY `index_time_entries_on_activity_id` (`activity_id`),
  KEY `index_time_entries_on_user_id` (`user_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci;
```

## 查询功能特性

### 支持的筛选条件

1. **项目维度**：按项目ID筛选，查看特定项目的工时
2. **任务维度**：按任务ID筛选，查看特定任务的工时
3. **人员维度**：按用户ID筛选，查看特定人员的工时
4. **活动类型**：按活动类型ID筛选，区分不同类型的工作
5. **日期范围**：按开始和结束日期筛选，支持任意时间段
6. **年月筛选**：快速查询指定年份或月份的工时记录

### 排序功能

支持按以下字段排序：
- `spent_on`：工作日期（默认）
- `created_on`：创建时间
- `hours`：工时大小

排序方向：
- `desc`：降序（默认）
- `asc`：升序

### 分页功能

- 默认每页 20 条记录
- 支持自定义每页大小
- 返回总记录数和总页数
