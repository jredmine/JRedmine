# 附件管理接口使用指南

## 概述

附件管理模块用于处理文件的上传、下载、查询和删除等操作。附件可以关联到不同的实体（如任务、项目、文档、Wiki页面等），采用多态关联设计。

## 接口列表

### 1. 上传附件

**接口地址：** `POST /api/attachments`

**权限要求：** 需要登录（`isAuthenticated()`）

**请求类型：** `multipart/form-data`

**请求参数：**

| 参数名        | 类型   | 必填 | 说明       | 示例                               |
| ------------- | ------ | ---- | ---------- | ---------------------------------- |
| file          | File   | 是   | 上传的文件 | -                                  |
| containerType | String | 否   | 容器类型   | Issue、Project、Document、WikiPage |
| containerId   | Long   | 否   | 容器ID     | 1                                  |
| description   | String | 否   | 文件描述   | 任务设计图                         |

**响应示例：**

```json
{
  "code": 200,
  "message": "附件上传成功",
  "data": {
    "id": 1,
    "containerType": "Issue",
    "containerId": 1,
    "filename": "design.png",
    "filesize": 102400,
    "contentType": "image/png",
    "downloads": 0,
    "author": {
      "id": 1,
      "login": "admin",
      "firstname": "管理",
      "lastname": "员"
    },
    "createdOn": "2026-01-18T10:30:00",
    "description": "任务设计图",
    "downloadUrl": "/api/attachments/1/download"
  },
  "success": true,
  "timestamp": "2026-01-18T10:30:00"
}
```

**使用示例：**

```bash
# 上传附件（关联到任务）
curl -X POST "http://localhost:8088/api/attachments" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@/path/to/file.png" \
  -F "containerType=Issue" \
  -F "containerId=1" \
  -F "description=任务设计图"

# 上传附件（不关联到任何实体）
curl -X POST "http://localhost:8088/api/attachments" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@/path/to/document.pdf"
```

### 2. 批量上传附件

**接口地址：** `POST /api/attachments/batch`

**权限要求：** 需要登录（`isAuthenticated()`）

**请求类型：** `multipart/form-data`

**请求参数：**

| 参数名        | 类型              | 必填 | 说明                               | 示例                               |
| ------------- | ----------------- | ---- | ---------------------------------- | ---------------------------------- |
| files         | File[]            | 是   | 上传的文件列表（多个文件）         | -                                  |
| containerType | String            | 否   | 容器类型                           | Issue、Project、Document、WikiPage |
| containerId   | Long              | 否   | 容器ID                             | 1                                  |
| description   | String            | 否   | 文件描述（所有文件共享此描述）     | 任务相关图片                       |

**响应示例：**

```json
{
  "code": 200,
  "message": "批量上传成功",
  "data": {
    "totalCount": 3,
    "successCount": 2,
    "failureCount": 1,
    "successes": [
      {
        "id": 1,
        "containerType": "Issue",
        "containerId": 15,
        "filename": "image1.png",
        "filesize": 102400,

        "downloads": 0,
        "author": {
          "id": 9,
          "login": "TangQF",
          "firstname": "Tang",
          "lastname": "QF"
        },
        "createdOn": "2026-01-18T10:30:00",
        "description": "任务相关图片",
        "downloadUrl": "/api/attachments/1/download"
      },
      {
        "id": 2,
        "containerType": "Issue",
        "containerId": 15,
        "filename": "image2.jpg",
        "filesize": 204800,
        "contentType": "image/jpeg",
        "downloads": 0,
        "author": {

        },
        "createdOn": "2026-01-18T10:30:01",
        "description": "任务相关图片",
        "downloadUrl": "/api/attachments/2/download"
      }
    ],
    "failures": [
      {
        "filename": "large-file.zip",
        "reason": "文件大小超过限制：10MB"
      }
    ]
  },
  "success": true
}
```

**使用示例：**

```bash
# 批量上传多个文件（所有文件共享相同的容器类型、容器ID和描述）
curl -X POST "http://localhost:8088/api/attachments/batch" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "files=@image1.png" \
  -F "files=@image2.jpg" \
  -F "files=@document.pdf" \
  -F "containerType=Issue" \
  -F "containerId=15" \
  -F "description=任务相关附件"

# 使用数组方式（某些客户端支持）
curl -X POST "http://localhost:8088/api/attachments/batch" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "files[]=@image1.png" \
  -F "files[]=@image2.jpg" \
  -F "containerType=Issue" \
  -F "containerId=15"
```

**注意事项：**

1. 所有文件共享相同的 `containerType`、`containerId` 和 `description`
2. 如果某个文件上传失败，不会影响其他文件的上传
3. 响应中包含成功和失败的详细信息
4. 如果所有文件都失败，返回错误响应（400）
5. 如果部分文件失败，返回部分成功响应（200）

### 4. 查询附件列表

**接口地址：** `GET /api/attachments`

**权限要求：** 需要登录（`isAuthenticated()`）

**说明：** 支持多条件筛选和分页查询，通过 `containerType` 和 `containerId` 参数可查询指定容器的附件

**查询参数：** （所有参数均为可选）

| 参数名        | 类型    | 说明                | 示例      |
| ------------- | ------- | ------------------- | --------- |
| containerType | String  | 容器类型            | Issue     |
| containerId   | Long    | 容器ID              | 1         |
| authorId      | Long    | 上传者ID            | 2         |
| filename      | String  | 文件名（模糊搜索）  | design    |
| contentType   | String  | 文件类型            | image/png |
| current       | Integer | 当前页码（从1开始） | 1         |
| size          | Integer | 每页大小            | 20        |

**响应示例：**

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "records": [
      {
        "id": 1,
        "containerType": "Issue",
        "containerId": 1,
        "filename": "design.png",
        "filesize": 102400,
        "contentType": "image/png",
        "downloads": 5,
        "author": {
          "id": 1,
          "login": "admin",
          "firstname": "管理",
          "lastname": "员"
        },
        "createdOn": "2026-01-18T10:30:00",
        "description": "任务设计图",
        "downloadUrl": "/api/attachments/1/download"
      }
    ],
    "total": 100,
    "current": 1,
    "size": 20,
    "pages": 5
  },
  "success": true
}
```

**使用示例：**

```bash
# 查询所有附件（分页）
curl -X GET "http://localhost:8088/api/attachments?current=1&size=20" \
  -H "Authorization: Bearer YOUR_TOKEN"

# 查询指定容器的附件
curl -X GET "http://localhost:8088/api/attachments?containerType=Issue&containerId=1" \
  -H "Authorization: Bearer YOUR_TOKEN"

# 按文件名模糊搜索
curl -X GET "http://localhost:8088/api/attachments?filename=design" \
  -H "Authorization: Bearer YOUR_TOKEN"

# 按文件类型搜索
curl -X GET "http://localhost:8088/api/attachments?contentType=image" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```bash
curl -X GET "http://localhost:8088/api/attachments?containerType=Project&containerId=5" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 5. 更新附件信息

**接口地址：** `PUT /api/attachments/{id}`

**权限要求：** 需要登录（`isAuthenticated()`），只能更新自己上传的附件或需要管理员权限

**路径参数：**

- `id` (Long): 附件ID

**请求参数：** （所有字段均为可选）

```json
{
  "description": "更新后的描述",
  "filename": "new-filename.png"
}
```

**响应示例：**

```json
{
  "code": 200,
  "message": "附件更新成功",
  "data": {
    "id": 1,
    "containerType": "Issue",
    "containerId": 1,
    "filename": "new-filename.png",
    "filesize": 102400,
    "contentType": "image/png",
    "downloads": 5,
    "author": {
      "id": 1,
      "login": "admin",
      "firstname": "管理",
      "lastname": "员"
    },
    "createdOn": "2026-01-18T10:30:00",
    "description": "更新后的描述",
    "downloadUrl": "/api/attachments/1/download"
  },
  "success": true
}
```

**使用示例：**

```bash
# 更新附件描述
curl -X PUT "http://localhost:8088/api/attachments/1" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "description": "更新后的描述"
  }'

# 更新附件文件名
curl -X PUT "http://localhost:8088/api/attachments/1" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "filename": "new-filename.png"
  }'
```

**注意事项：**

1. 只能更新附件的元数据（描述、文件名），不能更新文件内容
2. 只能更新自己上传的附件，管理员可以更新任何附件
3. 更新文件名不会改变磁盘上的实际文件名

### 6. 删除附件

**接口地址：** `DELETE /api/attachments/{id}`

**权限要求：** 需要登录（`isAuthenticated()`），只能删除自己上传的附件或需要管理员权限

**路径参数：**

- `id` (Long): 附件ID

**响应示例：**

```json
{
  "code": 200,
  "message": "附件删除成功",
  "data": null,
  "success": true
}
```

**使用示例：**

```bash
curl -X DELETE "http://localhost:8088/api/attachments/1" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

**注意事项：**

1. 删除操作会同时删除数据库记录和磁盘文件
2. 删除操作不可逆，请谨慎操作
3. 只能删除自己上传的附件，管理员可以删除任何附件

### 7. 下载附件

**接口地址：** `GET /api/attachments/{id}/download`

**权限要求：** 需要登录（`isAuthenticated()`）

**路径参数：**

- `id` (Long): 附件ID

**响应：** 文件流

**使用示例：**

```bash
# 下载附件
curl -X GET "http://localhost:8088/api/attachments/1/download" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -o downloaded-file.png

# 使用wget下载
wget --header="Authorization: Bearer YOUR_TOKEN" \
  "http://localhost:8088/api/attachments/1/download" \
  -O downloaded-file.png
```

**注意事项：**

1. 下载操作会自动增加下载次数
2. 响应头会包含正确的文件名和内容类型
3. 文件名会进行URL编码，确保支持中文等特殊字符

## 使用场景

### 场景 1：批量上传任务附件

一次性上传多个附件到同一个任务：

```bash
curl -X POST "http://localhost:8088/api/attachments/batch" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "files=@design1.png" \
  -F "files=@design2.png" \
  -F "files=@requirement.pdf" \
  -F "containerType=Issue" \
  -F "containerId=15" \
  -F "description=任务相关附件"
```

### 场景 2：为任务添加附件

用户在创建或编辑任务时，可以上传相关附件（如设计图、截图、文档等）：

```bash
# 1. 上传附件并关联到任务
curl -X POST "http://localhost:8088/api/attachments" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@design.png" \
  -F "containerType=Issue" \
  -F "containerId=1" \
  -F "description=UI设计图"

# 2. 查询任务的所有附件
curl -X GET "http://localhost:8088/api/attachments?containerType=Issue&containerId=1" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 场景 3：项目文档管理

项目经理可以为项目上传相关文档：

```bash
# 上传项目需求文档
curl -X POST "http://localhost:8088/api/attachments" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@requirement.pdf" \
  -F "containerType=Project" \
  -F "containerId=1" \
  -F "description=项目需求文档v1.0"

# 查询项目的所有附件
curl -X GET "http://localhost:8088/api/attachments?containerType=Project&containerId=1" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 场景 4：Wiki页面添加图片

用户在编辑Wiki页面时，可以上传图片：

```bash
# 上传图片
curl -X POST "http://localhost:8088/api/attachments" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@screenshot.png" \
  -F "containerType=WikiPage" \
  -F "containerId=1" \
  -F "description=使用说明截图"
```

### 场景 5：临时文件上传

用户可以先上传文件，稍后再关联到具体实体：

```bash
# 1. 上传文件（不关联）
curl -X POST "http://localhost:8088/api/attachments" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@document.pdf"

# 返回: {"id": 123, ...}

# 2. 稍后通过业务逻辑关联到具体实体（需要业务代码实现）
```

### 场景 6：批量下载附件

下载任务的所有附件：

```bash
# 1. 查询任务的所有附件
ATTACHMENTS=$(curl -X GET "http://localhost:8088/api/attachments?containerType=Issue&containerId=1" \
  -H "Authorization: Bearer YOUR_TOKEN")

# 2. 遍历并下载每个附件（需要脚本处理JSON）
# 示例：下载ID为1、2、3的附件
for id in 1 2 3; do
  curl -X GET "http://localhost:8088/api/attachments/${id}/download" \
    -H "Authorization: Bearer YOUR_TOKEN" \
    -o "attachment-${id}"
done
```

### 场景 7：文件管理与清理

管理员查看和管理系统中的所有附件：

```bash
# 查询所有附件
curl -X GET "http://localhost:8088/api/attachments?current=1&size=50" \
  -H "Authorization: Bearer YOUR_TOKEN"

# 查询某个用户上传的所有附件
curl -X GET "http://localhost:8088/api/attachments?authorId=2" \
  -H "Authorization: Bearer YOUR_TOKEN"

# 删除不需要的附件
curl -X DELETE "http://localhost:8088/api/attachments/123" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## 配置说明

### 文件上传配置

在 `application.yml` 中配置文件上传参数：

```yaml
spring:
  servlet:
    multipart:
      enabled: true
      max-file-size: 10MB      # 单个文件最大大小
      max-request-size: 50MB   # 整个请求最大大小
```

### 附件存储配置

在 `application-local.yml` 中配置附件存储参数：

```yaml
attachment:
  storage:
    path: files # 附件存储路径（相对于项目根目录）
  max-file-size: 10485760 # 最大文件大小（字节），默认10MB
  allowed-extensions: # 允许的文件扩展名，留空表示不限制
    # 例如: jpg,jpeg,png,gif,pdf,doc,docx,xls,xlsx,txt,zip,rar
```

**配置说明：**

- `storage.path`: 附件存储的根目录，相对于项目根目录
- `max-file-size`: 单个文件最大大小（字节），默认10MB（10485760字节）
- `allowed-extensions`: 允许上传的文件扩展名，用逗号分隔。留空表示不限制

## 文件存储结构

附件按日期分目录存储，结构如下：

```
files/
├── 2026/
│   ├── 01/
│   │   ├── uuid1.png
│   │   ├── uuid2.pdf
│   │   └── uuid3.docx
│   └── 02/
│       ├── uuid4.jpg
│       └── uuid5.xlsx
└── 2027/
    └── 01/
        └── uuid6.zip
```

**说明：**

- 文件按年/月分目录存储（如：`2026/01`）
- 文件名使用UUID生成，保证唯一性
- 数据库中记录了原始文件名、磁盘文件名和目录信息

## 数据库结构

附件数据存储在 `attachments` 表中：

```sql
CREATE TABLE `attachments` (
  `id` int NOT NULL AUTO_INCREMENT,
  `container_id` int DEFAULT NULL,           -- 容器ID（多态关联）
  `container_type` varchar(30) DEFAULT NULL, -- 容器类型（Issue、Project等）
  `filename` varchar(255) NOT NULL,          -- 原始文件名
  `disk_filename` varchar(255) NOT NULL,     -- 磁盘文件名（UUID）
  `filesize` bigint NOT NULL,                -- 文件大小（字节）
  `content_type` varchar(255) DEFAULT '',    -- 文件类型（MIME）
  `digest` varchar(64) NOT NULL,             -- 文件摘要（SHA-256）
  `downloads` int NOT NULL DEFAULT '0',      -- 下载次数
  `author_id` int NOT NULL,                  -- 上传者ID
  `created_on` timestamp NULL DEFAULT NULL,  -- 创建时间
  `description` varchar(255) DEFAULT NULL,   -- 文件描述
  `disk_directory` varchar(255) DEFAULT NULL,-- 磁盘目录
  PRIMARY KEY (`id`),
  KEY `index_attachments_on_author_id` (`author_id`),
  KEY `index_attachments_on_created_on` (`created_on`),
  KEY `index_attachments_on_container_id_and_container_type` (`container_id`,`container_type`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 容器类型说明

附件可以关联到以下类型的实体：

| 容器类型 | 说明     | 示例                     |
| -------- | -------- | ------------------------ |
| Issue    | 任务     | 任务的设计图、截图、文档 |
| Project  | 项目     | 项目需求文档、合同       |
| Document | 文档     | 文档的附件               |
| WikiPage | Wiki页面 | Wiki页面的图片、附件     |
| Message  | 论坛消息 | 论坛帖子的附件           |
| Version  | 版本     | 版本发布说明的附件       |

## 安全说明

1. **权限控制**：所有接口都需要登录才能访问
2. **作者验证**：只能修改和删除自己上传的附件（管理员除外）
3. **文件验证**：支持配置允许的文件类型和大小限制
4. **文件摘要**：使用SHA-256计算文件摘要，确保文件完整性
5. **文件隔离**：使用UUID作为磁盘文件名，避免文件名冲突和泄露

## 最佳实践

1. **合理设置文件大小限制**：根据实际需求设置 `max-file-size`，避免占用过多存储空间
2. **定期清理**：定期清理不再使用的附件，释放存储空间
3. **备份策略**：定期备份 `files` 目录和数据库
4. **文件类型限制**：根据安全需求设置 `allowed-extensions`，防止上传恶意文件
5. **监控存储空间**：监控文件存储空间使用情况，避免磁盘空间不足
6. **CDN集成**：对于高并发场景，可以考虑将文件存储到OSS并使用CDN加速
