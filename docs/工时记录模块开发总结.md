# 工时记录模块开发总结

## 概述

工时记录（Time Entry）模块是项目管理系统的核心功能之一，用于记录和管理团队成员在项目或任务上花费的工作时间。本文档总结了工时记录模块的完整开发过程和实现细节。

## 开发时间

**开发日期**：2026年1月12日  
**开发耗时**：约2小时

## 实现的功能

### 1. ✅ 创建工时记录

**接口**：`POST /api/time-entries`

**功能特性**：
- 支持关联任务或仅关联项目
- 支持为自己或其他成员记录工时（管理员）
- 自动计算年、月、周字段用于统计
- 自动设置记录创建者（authorId）
- 完整的数据验证（项目、任务、用户、活动类型）

### 2. ✅ 获取工时记录详情

**接口**：`GET /api/time-entries/{id}`

**功能特性**：
- 返回完整的工时记录信息
- 包含关联的项目、任务、用户、活动类型详细信息
- 展示创建者和工作人员信息

### 3. ✅ 查询工时记录列表

**接口**：`GET /api/time-entries`

**功能特性**：
- **8种筛选条件**：项目、任务、用户、活动类型、日期范围、年份、月份
- **3种排序字段**：工作日期、创建时间、工时大小
- **灵活的分页**：自定义每页大小
- **组合查询**：支持多条件组合筛选

### 4. ✅ 更新工时记录

**接口**：`PUT /api/time-entries/{id}`

**功能特性**：
- 部分更新：只传入需要修改的字段
- 权限控制：只能更新自己创建的记录
- 自动重算：更新日期时自动重新计算年、月、周
- 完整验证：验证所有关联数据的有效性

### 5. ✅ 删除工时记录

**接口**：`DELETE /api/time-entries/{id}`

**功能特性**：
- 权限控制：只能删除自己创建的记录
- 物理删除：从数据库永久删除
- 删除验证：确保操作成功

## 技术实现

### 数据模型

**核心实体**：`TimeEntry`

```java
@TableName("time_entries")
public class TimeEntry {
    @TableId(type = IdType.AUTO)
    private Long id;
    private Long projectId;      // 项目ID
    private Long authorId;       // 记录创建者ID
    private Long userId;         // 工作人员ID
    private Long issueId;        // 任务ID（可选）
    private Float hours;         // 工时（小时）
    private String comments;     // 备注
    private Long activityId;     // 活动类型ID
    private LocalDate spentOn;   // 工作日期
    private Integer tyear;       // 年份（冗余）
    private Integer tmonth;      // 月份（冗余）
    private Integer tweek;       // 周数（冗余）
    private LocalDateTime createdOn;
    private LocalDateTime updatedOn;
}
```

### 分层架构

```
Controller → Service → Mapper → Database
    ↓         ↓          ↓
   DTO    Business    MyBatis-Plus
          Logic
```

**1. Controller 层**
- `TimeEntryController`：REST API 接口
- 参数验证：使用 `@Valid` 注解
- 权限控制：使用 `@PreAuthorize` 注解

**2. Service 层**
- `TimeEntryService`：业务逻辑处理
- 数据验证：验证关联数据的有效性
- 权限检查：确保用户只能操作自己的数据
- 事务管理：使用 `@Transactional` 注解

**3. Mapper 层**
- `TimeEntryMapper`：数据访问层
- 继承 `BaseMapper<TimeEntry>`
- 使用 MyBatis-Plus 简化 CRUD 操作

### 权限设计

**权限模型**：
```java
// 创建工时记录
@PreAuthorize("hasRole('ADMIN') or authentication.principal.hasPermission('log_time')")

// 查看工时记录
@PreAuthorize("hasRole('ADMIN') or authentication.principal.hasPermission('view_time_entries')")

// 编辑/删除工时记录
@PreAuthorize("hasRole('ADMIN') or authentication.principal.hasPermission('edit_time_entries')")
```

**权限规则**：
1. 管理员（`ROLE_ADMIN`）拥有所有权限
2. 普通用户需要对应的项目权限
3. 只能修改/删除自己创建的工时记录（`authorId` 检查）

### 数据验证

**创建/更新时的验证**：
1. 项目是否存在
2. 任务是否存在且属于指定项目
3. 用户是否存在
4. 活动类型是否存在且类型正确
5. 工时必须大于0

### 查询优化

**索引设计**：
```sql
KEY `time_entries_project_id` (`project_id`)
KEY `time_entries_issue_id` (`issue_id`)
KEY `index_time_entries_on_activity_id` (`activity_id`)
KEY `index_time_entries_on_user_id` (`user_id`)
KEY `index_time_entries_on_created_on` (`created_on`)
```

**查询特性**：
- 使用 MyBatis-Plus 的 `LambdaQueryWrapper` 动态构建查询
- 支持多条件组合筛选
- 分页查询防止大数据量问题

## 使用场景

### 1. 日常工时记录
团队成员每日记录工作时间，包括任务和工作内容。

### 2. 项目工时统计
项目经理查询项目的总工时，用于成本核算和进度跟踪。

### 3. 个人工时汇总
团队成员查看自己的月度工时，用于绩效考核。

### 4. 工时报表导出
管理员导出指定时间段的工时数据，生成报表。

### 5. 错误记录修正
用户更新或删除记录错误的工时。

## 开发经验总结

### 1. 权限设计
- 使用 Spring Security 的 `@PreAuthorize` 注解
- 结合角色（Role）和权限（Permission）双重控制
- 在 Service 层再次验证数据所有权

### 2. 数据验证
- Controller 层使用 `@Valid` 进行基础验证
- Service 层进行业务逻辑验证
- 验证关联数据的有效性和一致性

### 3. 查询优化
- 使用 MyBatis-Plus 简化 CRUD 操作
- 使用 `LambdaQueryWrapper` 构建类型安全的查询
- 合理设计数据库索引

### 4. DTO 设计
- 请求和响应使用不同的 DTO
- 请求 DTO 包含验证注解
- 响应 DTO 包含关联数据的详细信息

### 5. 代码复用
- 提取公共的 DTO（如 `UserSimpleResponseDTO`）
- 提取公共的转换方法（如 `convertToResponseDTO`）
- 使用继承和组合减少重复代码

- 先设计 API 文档
- 明确请求和响应格式
- 编写使用示例和测试用例
