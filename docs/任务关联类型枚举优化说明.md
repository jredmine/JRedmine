# 任务关联类型枚举优化说明

## 问题描述

在创建任务关联关系时，`relationType` 字段使用的是 `String` 类型，没有进行枚举限制，可能导致：

1. ❌ 传入无效的关联类型值（如 "invalid_type"）
2. ❌ 拼写错误（如 "precede" 而不是 "precedes"）
3. ❌ 缺少类型安全检查
4. ❌ IDE无法提供代码提示
5. ❌ API文档中无法清晰展示可选值

## 解决方案

### 1. 创建枚举类 `IssueRelationType`

位置：`src/main/java/com/github/jredmine/enums/IssueRelationType.java`

**特点**：
- ✅ 定义了所有9种关联类型
- ✅ 包含code（数据库存储值）和description（中文描述）
- ✅ 支持JSON序列化/反序列化（@JsonValue, @JsonCreator）
- ✅ 提供业务方法（supportsDelay、isSymmetric、getReverse）
- ✅ 自动类型验证

**枚举值**：
```java
public enum IssueRelationType {
    RELATES("relates", "相关"),
    DUPLICATES("duplicates", "重复"),
    DUPLICATED("duplicated", "被重复"),
    BLOCKS("blocks", "阻塞"),
    BLOCKED("blocked", "被阻塞"),
    PRECEDES("precedes", "前置"),
    FOLLOWS("follows", "后置"),
    COPIED_TO("copied_to", "复制到"),
    COPIED_FROM("copied_from", "从...复制");
}
```

**业务方法**：

1. **supportsDelay()** - 是否支持延迟天数
   ```java
   if (delay != null && delay > 0 && !relationType.supportsDelay()) {
       throw new BusinessException("延迟天数仅用于 precedes/follows 类型");
   }
   ```

2. **isSymmetric()** - 是否为对称关系
   ```java
   // 用于未来实现自动反向关联
   if (relationType.isSymmetric()) {
       createReverseRelation(issueToId, issueFromId, relationType.getReverse());
   }
   ```

3. **getReverse()** - 获取反向关联类型
   ```java
   IssueRelationType reverse = IssueRelationType.BLOCKS.getReverse();
   // reverse = IssueRelationType.BLOCKED
   ```

### 2. 修改请求DTO

**修改前**：
```java
private String relationType;
```

**修改后**：
```java
@NotNull(message = "关联类型不能为空")
@Schema(description = "关联类型", 
        allowableValues = {"relates", "duplicates", "duplicated", "blocks", "blocked", 
                         "precedes", "follows", "copied_to", "copied_from"})
private IssueRelationType relationType;
```

### 3. 修改Service逻辑

**修改前**：
```java
String relationType = requestDTO.getRelationType();
if (relationType == null || relationType.trim().isEmpty()) {
    throw new BusinessException("关联类型不能为空");
}
if (delay != null && delay > 0 && 
    !relationType.equals("precedes") && !relationType.equals("follows")) {
    throw new BusinessException("延迟天数仅用于 precedes/follows 类型");
}
```

**修改后**：
```java
IssueRelationType relationType = requestDTO.getRelationType();
if (relationType == null) {
    throw new BusinessException("关联类型不能为空");
}
// 使用枚举方法，更简洁、更安全
if (delay != null && delay > 0 && !relationType.supportsDelay()) {
    throw new BusinessException("延迟天数仅用于 precedes/follows 类型");
}
```

## JSON序列化/反序列化

### 序列化（Java → JSON）

使用 `@JsonValue` 注解：
```java
@JsonValue
public String getCode() {
    return code;
}
```

**效果**：
```java
IssueRelationType type = IssueRelationType.PRECEDES;
// JSON输出: "precedes"
```

### 反序列化（JSON → Java）

使用 `@JsonCreator` 注解：
```java
@JsonCreator
public static IssueRelationType fromCode(String code) {
    for (IssueRelationType type : values()) {
        if (type.code.equals(code)) {
            return type;
        }
    }
    throw new IllegalArgumentException("Invalid IssueRelationType code: " + code);
}
```

**效果**：
```json
{ "relationType": "precedes" }
```
自动转换为 `IssueRelationType.PRECEDES`

## 未来扩展

有了枚举类型后，可以很方便地实现以下功能：

### 1. 自动反向关联

```java

    // 创建正向关联
    createRelation(issueFromId, issueToId, relationType);
    
    // 自动创建反向关联
    createRelation(issueToId, issueFromId, relationType.getReverse());
}
```

### 2. 关联类型分组

```java
// 按业务含义分组
public enum RelationGroup {

    REFERENCE(RELATES),
    COPY(COPIED_TO, COPIED_FROM);
}
```

```java
}
```
