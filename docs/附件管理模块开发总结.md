# 附件管理模块开发总结

## 概述

本次开发实现了完整的附件管理功能模块，支持文件上传、下载、查询、更新和删除等操作。附件采用多态关联设计，可以关联到任务(Issue)、项目(Project)、文档(Document)、Wiki页面等不同实体。

## 实现的功能

### 1. 核心功能
- ✅ 文件上传（支持多态关联）
- ✅ 文件下载（自动增加下载次数）
- ✅ 附件查询（支持多条件筛选和分页）
- ✅ 附件更新（更新描述和文件名）
- ✅ 附件删除（同时删除数据库记录和物理文件）
- ✅ 按容器查询附件列表

### 2. 安全特性
- ✅ 权限控制（需要登录才能访问）
- ✅ 作者验证（只能修改/删除自己上传的附件，管理员除外）
- ✅ 文件大小限制（可配置）
- ✅ 文件类型验证（可配置允许的扩展名）
- ✅ 文件完整性验证（SHA-256摘要）
- ✅ 文件名安全（使用UUID避免冲突）

### 3. 存储特性
- ✅ 按日期分目录存储（年/月）
- ✅ UUID文件名（避免冲突和泄露）
- ✅ 支持配置存储路径
- ✅ 自动创建存储目录

## 技术架构

### 1. 实体层（Entity）
- **Attachment.java** - 附件实体类
  - 支持多态关联（containerType + containerId）
  - 记录文件元数据（文件名、大小、类型等）
  - 记录存储信息（磁盘文件名、目录）
  - 记录统计信息（下载次数）

### 2. 数据访问层（Mapper）
- **AttachmentMapper.java** - 附件Mapper接口
  - 基于MyBatis-Plus的BaseMapper
  - 自定义方法：按容器查询、增加下载次数
- **AttachmentMapper.xml** - Mapper XML配置

### 3. 服务层（Service）
- **AttachmentService.java** - 附件服务类
  - 文件上传处理（验证、存储、记录）
  - 文件下载处理（验证、统计）
  - 附件查询（分页、筛选）
  - 附件管理（更新、删除）
  - 文件摘要计算（SHA-256）

### 4. 控制层（Controller）
- **AttachmentController.java** - 附件控制器
  - RESTful API接口
  - Swagger文档支持
  - 权限验证（Spring Security）
  - 文件下载响应处理

### 5. DTO层（Data Transfer Object）
- **请求DTO**:
  - AttachmentUploadRequestDTO - 上传请求
  - AttachmentQueryRequestDTO - 查询请求
  - AttachmentUpdateRequestDTO - 更新请求
  
- **响应DTO**:
  - AttachmentResponseDTO - 附件响应

## 接口清单

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 上传附件 | POST | /api/attachments | 上传文件并创建附件记录 |
| 获取附件详情 | GET | /api/attachments/{id} | 根据ID获取附件详细信息 |
| 查询附件列表 | GET | /api/attachments | 支持多条件筛选和分页 |
| 根据容器查询 | GET | /api/attachments/container/{type}/{id} | 查询指定容器的所有附件 |
| 更新附件信息 | PUT | /api/attachments/{id} | 更新附件的描述和文件名 |
| 删除附件 | DELETE | /api/attachments/{id} | 删除附件记录和文件 |
| 下载附件 | GET | /api/attachments/{id}/download | 下载附件文件 |

## 配置说明

### application.yml
```yaml
spring:
  servlet:
    multipart:
      enabled: true
      max-file-size: 10MB      # 单个文件最大大小
      max-request-size: 50MB   # 整个请求最大大小
```

### application-local.yml
```yaml
attachment:
  storage:
    path: files # 附件存储路径
  max-file-size: 10485760 # 最大文件大小（字节）
  allowed-extensions: # 允许的文件扩展名（留空表示不限制）
```

## 数据库表结构

**attachments表**包含以下字段：
- `id` - 主键
- `container_id`, `container_type` - 多态关联
- `filename` - 原始文件名
- `disk_filename` - 磁盘文件名（UUID）
- `disk_directory` - 磁盘目录（年/月）
- `filesize` - 文件大小
- `content_type` - 文件类型（MIME）
- `digest` - 文件摘要（SHA-256）
- `downloads` - 下载次数
- `author_id` - 上传者ID
- `created_on` - 创建时间
- `description` - 文件描述

## 使用示例

### 上传附件
```bash
curl -X POST "http://localhost:8088/api/attachments" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@/path/to/file.png" \
  -F "containerType=Issue" \
  -F "containerId=1" \
  -F "description=任务设计图"
```

### 下载附件
```bash
curl -X GET "http://localhost:8088/api/attachments/1/download" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -o downloaded-file.png
```

### 查询附件
```bash
# 查询任务的所有附件
curl -X GET "http://localhost:8088/api/attachments/container/Issue/1" \
  -H "Authorization: Bearer YOUR_TOKEN"
```
