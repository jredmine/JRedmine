# 实体类主键注解修复说明

## 问题描述

启动应用时报错：
```
Can not find table primary key in Class: "com.github.jredmine.entity.ProjectTracker"
Can not find table primary key in Class: "com.github.jredmine.entity.ProjectTemplateRole"
Can not find table primary key in Class: "com.github.jredmine.entity.RolesManagedRole"
```

## 问题原因

这三个实体类是多对多关系的中间表，使用了**联合主键**，但没有添加 `@TableId` 注解，导致 MyBatis-Plus 无法识别主键。

## 数据库表结构

### 1. projects_trackers（项目-跟踪器关联表）
```sql
CREATE TABLE `projects_trackers` (
  `project_id` int NOT NULL DEFAULT '0',
  `tracker_id` int NOT NULL DEFAULT '0',
  UNIQUE KEY `projects_trackers_unique` (`project_id`,`tracker_id`)
)
```
**联合主键**: `project_id` + `tracker_id`

### 2. project_template_roles（项目模板-角色关联表）
```sql
CREATE TABLE `project_template_roles` (
  `project_id` int NOT NULL DEFAULT '0',
  `role_id` int NOT NULL DEFAULT '0',
  UNIQUE KEY `project_template_roles_unique` (`project_id`,`role_id`)
)
```
**联合主键**: `project_id` + `role_id`

### 3. roles_managed_roles（角色管理关系表）
```sql
CREATE TABLE `roles_managed_roles` (
  `role_id` int NOT NULL,
  `managed_role_id` int NOT NULL,
  UNIQUE KEY `index_roles_managed_roles_on_role_id_and_managed_role_id` (`role_id`,`managed_role_id`)
)
```
**联合主键**: `role_id` + `managed_role_id`

## 解决方案

为联合主键的字段添加 `@TableId` 注解。

### 修复前

```java
@Data
@TableName("projects_trackers")
public class ProjectTracker {
    private Long projectId;
    private Long trackerId;
}
```

### 修复后

```java
@Data
@TableName("projects_trackers")
public class ProjectTracker {
    @TableId
    private Long projectId;
    
    @TableId
    private Long trackerId;
}
```

## 修复的文件

1. `entity/ProjectTracker.java` - 添加了 `@TableId` 注解
2. `entity/ProjectTemplateRole.java` - 添加了 `@TableId` 注解
3. `entity/RolesManagedRole.java` - 添加了 `@TableId` 注解

## MyBatis-Plus 联合主键说明

在 MyBatis-Plus 中处理联合主键的方式：

### 方式1：使用 @TableId（当前采用）
```java
@TableId
private Long field1;

@TableId
private Long field2;
```

**优点**：简单直接  
**适用场景**：仅用于简单的中间表关联，不涉及复杂的CRUD操作

### 方式2：使用复合主键类（更规范）
如果需要基于联合主键进行复杂的CRUD操作，建议创建复合主键类：

```java
@Data
public class ProjectTrackerKey implements Serializable {
    private Long projectId;
    private Long trackerId;
}

@Data
@TableName("projects_trackers")
public class ProjectTracker {
    @TableId
    private ProjectTrackerKey id;
}
```

**优点**：更符合JPA规范，支持复杂操作  
**缺点**：需要额外的类

## 当前项目的选择

由于这三个表在当前项目中主要用于：
- 关联查询
- 简单的新增和删除
- 不涉及复杂的更新操作

因此采用**方式1**（直接使用 `@TableId`）即可满足需求。

## 注意事项

1. **联合主键的顺序**：在 MyBatis-Plus 中，多个 `@TableId` 的顺序应与数据库中的主键定义顺序一致

2. **Mapper查询**：查询时需要同时指定所有主键字段
   ```java
   queryWrapper.eq(ProjectTracker::getProjectId, projectId)
                .eq(ProjectTracker::getTrackerId, trackerId);
   ```

3. **更新操作**：联合主键的更新比较复杂，通常建议删除后重新插入
