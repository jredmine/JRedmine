# 工时统计功能实现说明

## 概述

工时统计功能是工时记录模块的重要扩展，提供多维度的数据分析能力，帮助团队管理者和项目负责人更好地了解项目进展和资源使用情况。

**开发日期**：2026年1月13日  
**开发耗时**：约1小时

## 功能特性

### 1. 工时汇总统计

提供工时数据的整体概览，包括：
- **总工时**：所有记录的工时累加
- **记录总数**：符合条件的工时记录数量
- **平均工时**：单条记录的平均工时
- **最小工时**：单条记录的最小工时
- **最大工时**：单条记录的最大工时

### 2. 工时分组统计

支持按4种维度进行分组统计：

#### 2.1 按项目分组
- **用途**：查看各项目的工时占比
- **应用场景**：
  - 评估项目工作量
  - 识别工时消耗最多的项目
  - 辅助项目资源分配决策

#### 2.2 按用户分组
- **用途**：查看团队成员的工时分布
- **应用场景**：
  - 评估个人工作量
  - 识别工作负荷过重或过轻的成员
  - 辅助绩效考核和资源调配

#### 2.3 按活动类型分组
- **用途**：查看不同工作类型的工时占比
- **应用场景**：
  - 分析工时在开发、测试、设计等活动中的分布
  - 识别瓶颈环节
  - 优化工作流程

#### 2.4 按日期分组
- **用途**：查看工时的时间变化趋势
- **应用场景**：
  - 分析项目进度
  - 识别工作高峰期和低谷期
  - 辅助资源规划

### 3. 多条件筛选

所有统计功能都支持以下筛选条件：
- 项目ID
- 任务ID
- 用户ID
- 活动类型ID
- 日期范围（开始日期、结束日期）
- 年份
- 月份

### 核心算法

#### 1. 动态查询条件构建

```java
private LambdaQueryWrapper<TimeEntry> buildStatisticsQueryWrapper(
    TimeEntryStatisticsRequestDTO request) {
    LambdaQueryWrapper<TimeEntry> wrapper = new LambdaQueryWrapper<>();
    
    // 根据请求参数动态添加查询条件
    if (request.getProjectId() != null) {
        wrapper.eq(TimeEntry::getProjectId, request.getProjectId());
    }
    // ... 其他条件
    
    return wrapper;
}
```

#### 2. 分组统计算法

```java
private List<GroupStatistics> groupStatistics(
    List<TimeEntry> entries, String groupBy, Float totalHours) {
    
    // 1. 根据分组维度进行分组
    Map<String, List<TimeEntry>> groupMap = entries.stream()
        .collect(Collectors.groupingBy(e -> getGroupKey(e, groupBy)));
    
    // 2. 计算每组的统计数据
    List<GroupStatistics> groups = new ArrayList<>();
    for (Map.Entry<String, List<TimeEntry>> entry : groupMap.entrySet()) {
        Float groupHours = entry.getValue().stream()
            .map(TimeEntry::getHours)
            .reduce(0f, Float::sum);
        Float percentage = (groupHours / totalHours) * 100;
        
        groups.add(GroupStatistics.builder()
            .groupKey(entry.getKey())
            .groupName(getGroupName(groupBy, entry.getKey()))
            .hours(groupHours)
            .count((long) entry.getValue().size())
            .percentage(percentage)
            .build());
    }
    
    // 3. 按工时降序排序
    groups.sort((a, b) -> Float.compare(b.getHours(), a.getHours()));
    
    return groups;
}
```

#### 3. 数据精度处理

所有工时数据保留2位小数：
```java
Math.round(hours * 100) / 100f
```

### API接口

#### 1. 工时汇总统计

```
GET /api/time-entries/summary
```

**权限**：`view_time_entries`

**示例请求**：
```bash
# 查询项目1的工时汇总
GET /api/time-entries/summary?projectId=1

# 查询2026年1月的工时汇总
GET /api/time-entries/summary?year=2026&month=1

# 查询指定日期范围的工时汇总
GET /api/time-entries/summary?startDate=2026-01-01&endDate=2026-01-31
```

#### 2. 工时分组统计

```
GET /api/time-entries/statistics
```

**权限**：`view_time_entries`

**示例请求**：
```bash
# 按项目分组统计
GET /api/time-entries/statistics?groupBy=project

# 在项目1内按用户分组
GET /api/time-entries/statistics?projectId=1&groupBy=user

# 按活动类型分组统计本月工时
GET /api/time-entries/statistics?groupBy=activity&year=2026&month=1

# 按日期分组统计本月趋势
GET /api/time-entries/statistics?groupBy=date&year=2026&month=1
```

## 性能优化

### 1. 查询优化
- 使用数据库索引加速查询
- 只查询必要的字段
- 避免N+1查询问题

### 2. 计算优化
- 使用Stream API进行集合操作
- 一次性加载所有数据后在内存中分组计算
- 减少数据库交互次数

### 3. 数据精度
- 所有百分比和工时保留2位小数
- 避免浮点数计算误差
