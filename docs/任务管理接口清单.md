## 📊 数据库表结构分析

### 1. issues 表（任务主表）

```sql
CREATE TABLE `issues` (
  `id` int NOT NULL AUTO_INCREMENT,
  `tracker_id` int NOT NULL,                    -- 跟踪器ID（Bug、Feature等）
  `project_id` int NOT NULL,                    -- 项目ID
  `subject` varchar(255) NOT NULL DEFAULT '',    -- 任务标题
  `description` longtext,                       -- 任务描述
  `due_date` date DEFAULT NULL,                 -- 截止日期
  `category_id` int DEFAULT NULL,               -- 任务分类ID
  `status_id` int NOT NULL,                     -- 任务状态ID
  `assigned_to_id` int DEFAULT NULL,            -- 分配给用户ID
  `priority_id` int NOT NULL,                   -- 优先级ID
  `fixed_version_id` int DEFAULT NULL,           -- 修复版本ID
  `author_id` int NOT NULL,                     -- 创建者ID
  `lock_version` int NOT NULL DEFAULT '0',     -- 乐观锁版本号
  `created_on` timestamp NULL DEFAULT NULL,     -- 创建时间
  `updated_on` timestamp NULL DEFAULT NULL,     -- 更新时间
  `start_date` date DEFAULT NULL,               -- 开始日期
  `done_ratio` int NOT NULL DEFAULT '0',         -- 完成度（0-100）
  `estimated_hours` float DEFAULT NULL,         -- 预估工时
  `parent_id` int DEFAULT NULL,                 -- 父任务ID
  `root_id` int DEFAULT NULL,                   -- 根任务ID
  `lft` int DEFAULT NULL,                       -- 左值（树形结构）
  `rgt` int DEFAULT NULL,                       -- 右值（树形结构）
  `is_private` tinyint(1) NOT NULL DEFAULT '0', -- 是否私有
  `closed_on` datetime DEFAULT NULL,            -- 关闭时间
  PRIMARY KEY (`id`)
)
```

**关键字段说明：**
- `parent_id`, `root_id`, `lft`, `rgt`: 支持父子任务，使用嵌套集合模型（Nested Set Model）
- `lock_version`: 乐观锁，防止并发更新冲突
- `is_private`: 私有任务，只有项目成员可见
- `done_ratio`: 完成度，0-100，用于进度跟踪
- `estimated_hours`: 预估工时，用于时间管理

### 2. issue_statuses 表（任务状态）

```sql
CREATE TABLE `issue_statuses` (
  `id` int NOT NULL AUTO_INCREMENT,
  `name` varchar(30) NOT NULL DEFAULT '',        -- 状态名称
  `description` varchar(255) DEFAULT NULL,       -- 状态描述
  `is_closed` tinyint(1) NOT NULL DEFAULT '0',  -- 是否已关闭状态
  `position` int DEFAULT NULL,                   -- 排序位置
  `default_done_ratio` int DEFAULT NULL,         -- 默认完成度
  PRIMARY KEY (`id`)
)
```

**说明：**
- 状态由工作流管理模块管理
- 状态转换需要遵循工作流规则

### 3. issue_categories 表（任务分类）

```sql
CREATE TABLE `issue_categories` (
  `id` int NOT NULL AUTO_INCREMENT,
  `project_id` int NOT NULL DEFAULT '0',          -- 项目ID
  `name` varchar(60) NOT NULL DEFAULT '',       -- 分类名称
  `assigned_to_id` int DEFAULT NULL,            -- 默认分配给用户ID
  PRIMARY KEY (`id`)
)
```

**说明：**
- 分类是项目级别的，每个项目可以有自己的分类
- 分类可以设置默认指派人

### 4. issue_relations 表（任务关联）

```sql
CREATE TABLE `issue_relations` (
  `id` int NOT NULL AUTO_INCREMENT,
  `issue_from_id` int NOT NULL,                 -- 源任务ID
  `issue_to_id` int NOT NULL,                    -- 目标任务ID
  `relation_type` varchar(255) NOT NULL DEFAULT '', -- 关联类型
  `delay` int DEFAULT NULL,                     -- 延迟天数
  PRIMARY KEY (`id`)
)
```

**关联类型：**
- `relates` - 相关
- `duplicates` - 重复
- `duplicated` - 被重复
- `blocks` - 阻塞
- `blocked` - 被阻塞
- `precedes` - 前置（用于甘特图）
- `follows` - 后置（用于甘特图）
- `copied_to` - 复制到
- `copied_from` - 从...复制

### 5. watchers 表（关注者）

```sql
CREATE TABLE `watchers` (
  `id` int NOT NULL AUTO_INCREMENT,
  `watchable_type` varchar(255) NOT NULL DEFAULT '', -- 可关注类型（'Issue'）
  `watchable_id` int NOT NULL DEFAULT '0',      -- 可关注对象ID
  `user_id` int DEFAULT NULL,                   -- 用户ID
  PRIMARY KEY (`id`)
)
```

**说明：**
- 支持多态关联，可以关注任务、文档等
- 关注者会收到任务更新通知

### 6. journals 表（活动日志）

```sql
CREATE TABLE `journals` (
  `id` int NOT NULL AUTO_INCREMENT,
  `journalized_id` int NOT NULL DEFAULT '0',     -- 被记录对象ID
  `journalized_type` varchar(30) NOT NULL DEFAULT '', -- 被记录类型（'Issue'）
  `user_id` int NOT NULL DEFAULT '0',           -- 操作用户ID
  `notes` longtext,                             -- 备注/评论
  `created_on` datetime NOT NULL,               -- 创建时间
  `updated_on` datetime DEFAULT NULL,          -- 更新时间
  `updated_by_id` int DEFAULT NULL,            -- 更新者ID
  `private_notes` tinyint(1) NOT NULL DEFAULT '0', -- 是否私有备注
  PRIMARY KEY (`id`)
)
```

**说明：**
- 记录任务的所有变更历史
- 支持评论功能（notes 字段）
- 支持私有备注（只有项目成员可见）

### 7. journal_details 表（活动详情）

```sql
CREATE TABLE `journal_details` (
  `id` int NOT NULL AUTO_INCREMENT,
  `journal_id` int NOT NULL DEFAULT '0',        -- 活动日志ID
  `property` varchar(30) NOT NULL DEFAULT '',   -- 属性名（'attr', 'attachment'等）
  `prop_key` varchar(30) NOT NULL DEFAULT '',   -- 属性键（字段名）
  `old_value` text,                            -- 旧值
  `new_value` text,                            -- 新值
  PRIMARY KEY (`id`)
)
```

**说明：**
- 记录字段变更的详细信息
- `property = 'attr'` 表示属性变更
- `property = 'attachment'` 表示附件变更

### 8. enumerations 表（枚举值）

```sql
CREATE TABLE `enumerations` (
  `id` int NOT NULL AUTO_INCREMENT,
  `name` varchar(30) NOT NULL DEFAULT '',        -- 名称
  `position` int DEFAULT NULL,                  -- 排序位置
  `is_default` tinyint(1) NOT NULL DEFAULT '0', -- 是否默认
  `type` varchar(255) DEFAULT NULL,             -- 类型（'IssuePriority'等）
  `active` tinyint(1) NOT NULL DEFAULT '1',     -- 是否激活
  `project_id` int DEFAULT NULL,                -- 项目ID（NULL表示全局）
  `parent_id` int DEFAULT NULL,                 -- 父ID
  `position_name` varchar(30) DEFAULT NULL,     -- 位置名称
  PRIMARY KEY (`id`)
)
```

**说明：**
- 用于存储优先级、活动类型等枚举值
- `type = 'IssuePriority'` 表示任务优先级
- 支持项目级别的枚举值（`project_id` 不为 NULL）

## 🔴 核心接口（P0 - 必须实现）

### 1. 获取任务列表 ⚠️

**接口**: `GET /api/issues`

**权限**: 需要 `view_issues` 权限

**功能说明**:
- 分页查询任务列表
- 支持多条件筛选（项目、状态、跟踪器、优先级、指派人、创建者等）
- 支持关键词搜索（标题、描述）
- 支持排序（创建时间、更新时间、优先级等）
- 私有任务仅项目成员可见

**请求参数**:
- `current` (Integer, 可选): 当前页码，默认 1
- `size` (Integer, 可选): 每页数量，默认 10
- `projectId` (Long, 可选): 项目ID
- `statusId` (Integer, 可选): 状态ID
- `trackerId` (Integer, 可选): 跟踪器ID
- `priorityId` (Integer, 可选): 优先级ID
- `assignedToId` (Long, 可选): 指派人ID
- `authorId` (Long, 可选): 创建者ID
- `categoryId` (Integer, 可选): 分类ID
- `fixedVersionId` (Integer, 可选): 版本ID
- `keyword` (String, 可选): 关键词（搜索标题和描述）
- `isPrivate` (Boolean, 可选): 是否私有
- `sortBy` (String, 可选): 排序字段（`created_on`, `updated_on`, `priority`, `due_date`等）
- `sortOrder` (String, 可选): 排序方向（`asc`, `desc`），默认 `desc`

**响应**: `PageResponse<IssueListItemResponseDTO>`

---

### 2. 获取任务详情 ⚠️

**接口**: `GET /api/issues/{id}`

**权限**: 需要 `view_issues` 权限

**功能说明**:
- 根据任务ID查询任务详细信息
- 包含任务的所有字段
- 包含关联信息（父任务、子任务、关联任务）
- 包含关注者列表
- 包含活动日志（变更历史）

**路径参数**:
- `id` (Long): 任务ID

**响应**: `IssueDetailResponseDTO`

---

### 3. 创建任务 ⚠️

**接口**: `POST /api/issues`

**权限**: 需要 `add_issues` 权限

**功能说明**:
- 创建新任务
- 自动设置创建者为当前用户
- 自动设置默认状态（跟踪器的默认状态）
- 支持设置父任务（创建子任务）
- 需要验证工作流规则（状态转换是否允许）

**请求体**: `IssueCreateRequestDTO`

**字段说明**:
- `projectId` (Long, 必填): 项目ID
- `trackerId` (Integer, 必填): 跟踪器ID
- `subject` (String, 必填): 任务标题
- `description` (String, 可选): 任务描述
- `statusId` (Integer, 可选): 状态ID（默认使用跟踪器的默认状态）
- `priorityId` (Integer, 必填): 优先级ID
- `assignedToId` (Long, 可选): 指派人ID
- `categoryId` (Integer, 可选): 分类ID
- `fixedVersionId` (Integer, 可选): 版本ID
- `startDate` (Date, 可选): 开始日期
- `dueDate` (Date, 可选): 截止日期
- `estimatedHours` (Float, 可选): 预估工时
- `doneRatio` (Integer, 可选): 完成度（0-100），默认 0
- `parentId` (Long, 可选): 父任务ID
- `isPrivate` (Boolean, 可选): 是否私有，默认 false

**响应**: `IssueDetailResponseDTO`

---

### 4. 更新任务 ⚠️

**接口**: `PUT /api/issues/{id}`

**权限**: 需要 `edit_issues` 权限

**功能说明**:
- 更新任务信息
- 支持部分更新（只更新提供的字段）
- 需要验证工作流规则（状态转换是否允许）
- 需要验证字段规则（必填、只读等）
- 使用乐观锁防止并发更新冲突
- 自动记录变更历史

**路径参数**:
- `id` (Long): 任务ID

**请求体**: `IssueUpdateRequestDTO`

**字段说明**: 同创建接口，所有字段可选

**响应**: `IssueDetailResponseDTO`

---

### 5. 删除任务 ⚠️

**接口**: `DELETE /api/issues/{id}`

**权限**: 需要 `delete_issues` 权限

**功能说明**:
- 删除任务（物理删除）
- 如果任务有子任务，需要先删除子任务
- 删除关联的任务关联关系
- 删除关注者记录
- 删除活动日志（可选，建议保留）

**路径参数**:
- `id` (Long): 任务ID

**响应**: `ApiResponse<Void>`

---

### 6. 更新任务状态 ⚠️

**接口**: `PUT /api/issues/{id}/status`

**权限**: 需要 `edit_issues` 权限

**功能说明**:
- 更新任务状态
- 需要验证工作流规则（状态转换是否允许）
- 需要验证用户角色权限
- 需要验证用户限制（创建者、指派人）
- 自动记录状态变更历史
- 如果状态是关闭状态，自动设置 `closed_on` 和 `done_ratio = 100`

**路径参数**:
- `id` (Long): 任务ID

**请求体**: `IssueStatusUpdateRequestDTO`
- `statusId` (Integer, 必填): 新状态ID
- `notes` (String, 可选): 备注/评论

**响应**: `IssueDetailResponseDTO`

---

### 7. 分配任务 ⚠️

**接口**: `PUT /api/issues/{id}/assign`

**权限**: 需要 `edit_issues` 权限

**功能说明**:
- 分配任务给指定用户
- 支持取消分配（`assignedToId = null`）
- 自动记录分配变更历史
- 可以发送通知给新指派人

**路径参数**:
- `id` (Long): 任务ID

**请求体**: `IssueAssignRequestDTO`
- `assignedToId` (Long, 可选): 指派人ID（null 表示取消分配）

**响应**: `IssueDetailResponseDTO`

---

### 8. 获取任务子任务列表 ⚠️

**接口**: `GET /api/issues/{id}/children`

**权限**: 需要 `view_issues` 权限

**功能说明**:
- 查询任务的所有子任务
- 支持递归查询（包含子任务的子任务）

**路径参数**:
- `id` (Long): 任务ID

**请求参数**:
- `recursive` (Boolean, 可选): 是否递归查询，默认 false

**响应**: `List<IssueListItemResponseDTO>`

---

### 9. 添加任务关联 ⚠️

**接口**: `POST /api/issues/{id}/relations`

**权限**: 需要 `edit_issues` 权限

**功能说明**:
- 添加任务关联关系
- 支持多种关联类型（相关、重复、阻塞、前置等）
- 支持设置延迟天数（用于甘特图）

**路径参数**:
- `id` (Long): 源任务ID

**请求体**: `IssueRelationCreateRequestDTO`
- `targetIssueId` (Long, 必填): 目标任务ID
- `relationType` (String, 必填): 关联类型（`relates`, `duplicates`, `blocks`, `precedes`等）
- `delay` (Integer, 可选): 延迟天数（仅用于 `precedes`/`follows` 类型）

**响应**: `IssueRelationResponseDTO`

---

### 10. 删除任务关联 ⚠️

**接口**: `DELETE /api/issues/{id}/relations/{relationId}`

**权限**: 需要 `edit_issues` 权限

**功能说明**:
- 删除任务关联关系

**路径参数**:
- `id` (Long): 任务ID
- `relationId` (Long): 关联ID

**响应**: `ApiResponse<Void>`

---

### 11. 添加任务关注者 ⚠️

**接口**: `POST /api/issues/{id}/watchers`

**权限**: 需要 `view_issues` 权限

**功能说明**:
- 添加任务关注者
- 关注者会收到任务更新通知
- 可以关注自己

**路径参数**:
- `id` (Long): 任务ID

**请求体**: `IssueWatcherCreateRequestDTO`
- `userId` (Long, 必填): 用户ID

**响应**: `ApiResponse<Void>`

---

### 12. 删除任务关注者 ⚠️

**接口**: `DELETE /api/issues/{id}/watchers/{userId}`

**权限**: 需要 `view_issues` 权限

**功能说明**:
- 删除任务关注者
- 只能删除自己的关注，或需要 `edit_issues` 权限

**路径参数**:
- `id` (Long): 任务ID
- `userId` (Long): 用户ID

**响应**: `ApiResponse<Void>`

## 🔒 权限说明

任务管理相关的权限（来自 `Permission` 枚举）：

- `view_issues` - 查看任务
- `add_issues` - 创建任务
- `edit_issues` - 编辑任务（包括添加评论）
- `delete_issues` - 删除任务
- `view_private_issues` - 查看私有任务
- `manage_issues` - 管理任务（包括分配、状态转换、优先级等）

**注意**: 任务分类管理权限 `manage_categories` 目前不在 `Permission` 枚举中，可以使用 `manage_projects` 权限或后续添加。

**权限验证规则：**
1. 私有任务（`is_private = true`）只有项目成员可见
2. 状态转换需要遵循工作流规则
3. 字段更新需要遵循工作流字段规则
4. 删除任务需要检查是否有子任务

## ⚠️ 注意事项

1. **乐观锁**: 使用 `lock_version` 字段防止并发更新冲突
2. **树形结构**: 使用嵌套集合模型（Nested Set Model）高效查询父子任务
3. **工作流验证**: 状态转换必须验证工作流规则
4. **权限控制**: 私有任务需要特殊权限检查
5. **变更历史**: 所有字段变更都需要记录到 `journals` 和 `journal_details` 表
6. **关联关系**: 任务关联支持多种类型，需要正确验证和存储
7. **性能优化**: 任务列表查询需要优化，考虑使用索引和分页
