# 系统设置模块 API 使用指南

## 概述

系统设置模块提供了对系统全局配置的管理功能，支持多种分类的配置项，包括基本设置、邮件配置、安全设置、附件设置和通知设置。

## 功能特性

- ✅ **分类管理**：按功能分类组织配置项
- ✅ **缓存机制**：内存缓存提升性能
- ✅ **默认值支持**：未配置项自动使用默认值
- ✅ **邮件测试**：验证邮件配置是否正确
- ✅ **重置功能**：一键恢复默认配置
- ✅ **权限控制**：仅管理员可访问

## 设置分类

系统设置共分为5个主要分类：

### 1. 基本设置 (general)
- **站点名称** (`app_title`): 系统显示的站点标题
- **站点描述** (`app_description`): 系统描述信息
- **默认语言** (`default_language`): 系统默认语言（zh-CN）
- **时区** (`time_zone`): 系统时区设置

### 2. 邮件设置 (email)
- **邮件发送启用** (`email_delivery`): 是否启用邮件发送
- **SMTP服务器** (`smtp_address`): SMTP服务器地址
- **SMTP端口** (`smtp_port`): SMTP服务器端口
- **SMTP用户名** (`smtp_username`): SMTP认证用户名
- **SMTP密码** (`smtp_password`): SMTP认证密码
- **SMTP认证方式** (`smtp_authentication`): 认证方式（plain/login）
- **启用TLS** (`smtp_enable_starttls_auto`): 是否启用TLS加密
- **发件人邮箱** (`emails_from`): 系统发送邮件的发件人地址

### 3. 安全设置 (security)
- **密码最小长度** (`password_min_length`): 用户密码最小字符数
- **密码必须包含数字** (`password_require_digit`): 密码复杂度要求
- **密码必须包含字母** (`password_require_letter`): 密码复杂度要求
- **密码必须包含特殊字符** (`password_require_special`): 密码复杂度要求
- **会话超时时间** (`session_timeout`): 用户会话超时时间（分钟）
- **登录失败最大次数** (`login_max_attempts`): 账户锁定前的最大失败次数
- **登录锁定时间** (`login_lock_time`): 账户锁定持续时间（分钟）

### 4. 附件设置 (attachment)
- **附件最大大小** (`attachment_max_size`): 单个附件最大大小（MB）
- **允许的文件扩展名** (`attachment_extensions_allowed`): 允许上传的文件类型
- **附件存储路径** (`attachments_storage_path`): 附件在服务器上的存储目录
- **缩略图启用** (`thumbnails_enabled`): 是否生成图片缩略图

### 5. 通知设置 (notification)
- **通知任务作者** (`notified_events_author`): 任务变更时是否通知作者
- **通知任务指派人** (`notified_events_assignee`): 任务变更时是否通知指派人
- **通知关注者** (`notified_events_watchers`): 任务变更时是否通知关注者

## API 接口清单

### 1. 获取所有系统设置

```http
GET /api/settings
Authorization: Bearer {token}
```

**权限要求**: 管理员

**响应示例**:
```json
{
  "code": 200,
  "message": "操作成功",
  "data": [
    {
      "category": "general",
      "categoryName": "基本设置",
      "settings": [
        {
          "id": 1,
          "name": "app_title",
          "value": "我的项目管理系统",
          "description": "站点名称",
          "category": "general",
          "defaultValue": "JRedmine",
          "updatedOn": "2026-01-11T10:30:00"
        }
      ]
    },
    {
      "category": "email",
      "categoryName": "邮件设置",
      "settings": [...]
    }
  ]
}
```

### 2. 根据分类获取设置

```http
GET /api/settings/category/{category}
Authorization: Bearer {token}
```

**路径参数**:
- `category`: 分类代码（general/email/security/attachment/notification）

**权限要求**: 管理员

**请求示例**:

## 技术实现要点

### 1. 枚举驱动的配置管理

使用 `SettingKey` 枚举定义所有支持的配置项，确保类型安全：

```java
@Getter
public enum SettingKey {
    SITE_TITLE("app_title", "站点名称", "JRedmine", SettingCategory.GENERAL),
    // ...
}
```

**优点**：
- ✅ 类型安全，避免拼写错误
- ✅ 自动提供默认值
- ✅ 清晰的分类结构
- ✅ IDE自动补全支持

### 2. 内存缓存机制

使用 `Map<String, String>` 实现简单的内存缓存：

```java
private final Map<String, String> settingCache = new HashMap<>();

public String getSetting(String name) {
    // 先从缓存获取
    if (settingCache.containsKey(name)) {
        return settingCache.get(name);
    }
    // 查询数据库...
}
```

**优点**：
- ✅ 减少数据库查询
- ✅ 提升响应速度
- ✅ 支持手动清除

### 3. 默认值回退机制

未配置的项自动使用枚举中定义的默认值：

```java
if (setting != null) {
    value = setting.getValue();
} else {
    SettingKey settingKey = SettingKey.fromKey(name);
    value = settingKey != null ? settingKey.getDefaultValue() : null;
}
```

### 4. 分类查询优化

按分类分组返回，便于前端展示：

```java
Map<SettingKey.SettingCategory, List<SettingResponseDTO>> groupedSettings = new HashMap<>();
```

## 注意事项

1. **权限控制**: 所有接口都需要管理员权限（`@PreAuthorize("hasRole('ADMIN')")`）

2. **敏感信息**: SMTP密码等敏感信息建议：
   - 使用环境变量
   - 加密存储
   - 限制访问权限

3. **缓存一致性**: 更新设置后自动更新缓存，多实例部署需要考虑缓存同步

4. **邮件测试**: 测试前确保：
   - SMTP配置正确
   - 网络连接正常
   - 防火墙规则允许

5. **默认值**: 删除设置不会物理删除记录，而是返回默认值

## 错误码说明

| 错误码 | 说明 | 解决方案 |
|--------|------|----------|
| 400 | 参数无效 | 检查设置项名称是否正确 |
| 401 | 未授权 | 提供有效的认证令牌 |
| 403 | 无权限 | 需要管理员权限 |
| 500 | 服务器错误 | 查看服务器日志 |

## 扩展开发

### 添加新的配置项

1. 在 `SettingKey` 枚举中添加新项：
```java
NEW_SETTING("new_setting", "新配置", "default_value", SettingCategory.GENERAL)
```

2. 无需修改其他代码，系统自动识别

### 添加新的配置分类

1. 在 `SettingKey.SettingCategory` 中添加新分类：
```java
NEW_CATEGORY("新分类", "new_category")
```

2. 添加属于该分类的配置项
