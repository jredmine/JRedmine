## 测试环境准备

### 1. 启动应用
```bash
cd /Users/panfeng/JavaProjects/JRedmine
mvn spring-boot:run
```

### 2. 获取认证Token

首先需要登录获取Token：

```bash
curl -X POST "http://localhost:8080/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{
```

## 自动化测试脚本

使用以下Shell脚本进行批量测试：

```bash
#!/bin/bash

# 配置
BASE_URL="http://localhost:8080"
TOKEN=""

# 登录获取Token
echo "登录中..."
response=$(curl -s -X POST "$BASE_URL/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"login":"admin","password":"admin"}')
TOKEN=$(echo $response | jq -r '.data.token')

if [ -z "$TOKEN" ]; then
  echo "登录失败"
  exit 1
fi

echo "Token: $TOKEN"
echo ""

# 测试1：基本查询
echo "测试1：基本查询"
curl -s -X GET "$BASE_URL/api/issues/gantt?projectId=1" \
  -H "Authorization: Bearer $TOKEN" | jq '.data.totalCount'
echo ""

# 测试2：包含子项目
echo "测试2：包含子项目"
curl -s -X GET "$BASE_URL/api/issues/gantt?projectId=1&includeSubprojects=true" \
  -H "Authorization: Bearer $TOKEN" | jq '.data.totalCount'
echo ""

# 测试3：只显示有日期的任务
echo "测试3：只显示有日期的任务"
curl -s -X GET "$BASE_URL/api/issues/gantt?projectId=1&onlyWithDates=true" \
  -H "Authorization: Bearer $TOKEN" | jq '.data.totalCount'
echo ""

# 测试4：按版本筛选
echo "测试4：按版本筛选"
curl -s -X GET "$BASE_URL/api/issues/gantt?projectId=1&versionId=1" \
  -H "Authorization: Bearer $TOKEN" | jq '.data.totalCount'
echo ""

# 测试5：参数验证
echo "测试5：参数验证（缺少projectId）"
curl -s -X GET "$BASE_URL/api/issues/gantt" \
  -H "Authorization: Bearer $TOKEN" | jq '.code'
echo ""

echo "所有测试完成！"
```

## 使用Postman测试

### 1. 导入Collection

创建一个Postman Collection，包含以下请求：

- **获取Token**: POST `/api/auth/login`
- **基本甘特图查询**: GET `/api/issues/gantt?projectId=1`
- **包含子项目**: GET `/api/issues/gantt?projectId=1&includeSubprojects=true`
- **按版本筛选**: GET `/api/issues/gantt?projectId=1&versionId=1`
- **只显示有日期的任务**: GET `/api/issues/gantt?projectId=1&onlyWithDates=true`

### 2. 设置环境变量

```json
{
  "baseUrl": "http://localhost:8080",
  "token": "{{token}}"
}
```

### 3. 自动设置Token

在登录请求的Tests中添加：

```javascript
pm.test("登录成功", function () {
    var jsonData = pm.response.json();
    pm.expect(jsonData.code).to.eql(200);
    pm.environment.set("token", jsonData.data.token);
});
```
