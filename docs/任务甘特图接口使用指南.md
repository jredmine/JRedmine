# 任务甘特图接口使用指南

## 概述

任务甘特图接口用于获取项目任务的甘特图数据，包括任务的开始日期、截止日期、完成度、依赖关系等信息，方便前端展示项目任务的时间线和进度。

## 接口信息

- **请求路径**: `GET /api/issues/gantt`
- **权限要求**: 需要登录认证，需要 `view_issues` 权限或系统管理员
- **返回格式**: JSON

## 请求参数

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| projectId | Long | 是 | 项目ID | 1 |
| includeSubprojects | Boolean | 否 | 是否包含子项目的任务，默认false | true |
| versionId | Long | 否 | 版本ID（筛选特定版本的任务），传0或null表示不筛选 | 1 |
| trackerId | Integer | 否 | 跟踪器ID（筛选特定跟踪器的任务），传0或null表示不筛选 | 1 |
| statusId | Integer | 否 | 状态ID（筛选特定状态的任务），传0或null表示不筛选 | 1 |
| assignedToId | Long | 否 | 分配给用户ID（筛选特定用户的任务），传0或null表示不筛选 | 1 |
| startDateFrom | LocalDate | 否 | 开始日期筛选（格式：YYYY-MM-DD） | 2026-01-01 |
| dueDateTo | LocalDate | 否 | 截止日期筛选（格式：YYYY-MM-DD） | 2026-12-31 |
| onlyWithDates | Boolean | 否 | 是否仅显示有日期的任务，默认false | true |

**注意**：对于ID类型的筛选参数（versionId、trackerId、statusId、assignedToId），传入0或null都表示不对该字段进行筛选。只有传入大于0的值时才会应用筛选条件。

## 使用示例

### 示例1：获取项目的所有任务甘特图

```bash
curl -X GET "http://localhost:8080/api/issues/gantt?projectId=1" \
  -H "Authorization: Bearer {token}"
```

### 示例2：获取项目及其子项目的所有任务

```bash
curl -X GET "http://localhost:8080/api/issues/gantt?projectId=1&includeSubprojects=true" \
  -H "Authorization: Bearer {token}"
```

### 示例3：获取特定版本的任务甘特图

```bash
curl -X GET "http://localhost:8080/api/issues/gantt?projectId=1&versionId=1" \
  -H "Authorization: Bearer {token}"
```

### 示例4：获取特定时间范围内的任务

```bash
curl -X GET "http://localhost:8080/api/issues/gantt?projectId=1&startDateFrom=2026-01-01&dueDateTo=2026-12-31" \
  -H "Authorization: Bearer {token}"
```

### 示例5：只获取有日期的任务

```bash
curl -X GET "http://localhost:8080/api/issues/gantt?projectId=1&onlyWithDates=true" \
  -H "Authorization: Bearer {token}"
```

### 示例6：获取指派给特定用户的任务

```bash
curl -X GET "http://localhost:8080/api/issues/gantt?projectId=1&assignedToId=1" \
  -H "Authorization: Bearer {token}"
```

## 权限说明

1. **访问权限**：
   - 需要认证（携带有效的Bearer Token）
   - 需要系统管理员权限或在项目中拥有 `view_issues` 权限

2. **私有任务可见性**：
   - 私有任务仅项目成员可见
   - 非项目成员只能看到非私有任务

3. **子项目权限**：
   - 如果包含子项目（includeSubprojects=true），会查询所有子项目的任务
   - 用户需要在相应子项目中拥有查看权限

## 业务规则

1. **任务排序**：
   - 按任务的树形结构（rootId, lft）排序
   - 保证父任务在子任务之前

2. **里程碑判断**：
   - 顶级任务（parentId为null）被标记为里程碑（isMilestone=true）

3. **依赖关系**：
   - 只返回precedes、follows、blocks、blocked四种类型的依赖
   - 依赖关系包含延迟天数（delay）

4. **甘特图时间范围**：
   - ganttStartDate：所有任务中最早的开始日期
   - ganttEndDate：所有任务中最晚的截止日期
   - 如果所有任务都没有日期，这两个字段为null

## 错误码说明

| 错误码 | 说明 |
|--------|------|
| 401 | 未认证或token无效 |
| 403 | 无权限访问（缺少view_issues权限） |
| 404 | 项目不存在 |
| 500 | 服务器内部错误 |

## 前端集成建议

### 1. 甘特图展示

可以使用以下JavaScript库来展示甘特图：

- **DHTMLX Gantt**: 功能强大的专业甘特图库
- **Frappe Gantt**: 轻量级、易用的甘特图库
- **FullCalendar**: 支持任务时间线展示

### 2. 数据转换示例（Frappe Gantt）

```javascript
// 将API返回的数据转换为Frappe Gantt格式
function convertToFrappeGantt(ganttData) {
  return ganttData.tasks.map(task => ({
    id: task.id.toString(),
    name: task.subject,
    start: task.startDate || ganttData.ganttStartDate,
    end: task.dueDate || ganttData.ganttEndDate,
    progress: task.doneRatio,
    dependencies: task.dependencies
      .filter(dep => dep.relationType === 'precedes')
      .map(dep => dep.issueId.toString())
      .join(','),
    custom_class: task.statusClosed ? 'bar-closed' : ''
  }));
}

// 使用示例
const gantt = new Gantt("#gantt", tasks, {
  view_mode: 'Week',
  language: 'zh-CN'
});
```

### 3. 颜色标识建议

- **状态**：已关闭的任务使用灰色，进行中的任务使用蓝色
- **优先级**：紧急任务使用红色，普通任务使用绿色
- **完成度**：使用进度条显示（0-100%）
- **里程碑**：使用菱形图标标识

### 4. 交互功能

- 点击任务跳转到任务详情页
- 支持拖拽调整任务时间
- 支持任务依赖关系的可视化连线
- 支持缩放和滚动查看更多任务

## 注意事项

1. **性能考虑**：
   - 如果项目任务数量很大（>1000），建议使用筛选条件减少返回数据量
   - 考虑添加分页功能（未来版本可能支持）

2. **日期处理**：
   - 部分任务可能没有开始日期或截止日期
   - 前端需要处理这种情况，可以使用默认日期或不显示

3. **依赖关系**：
   - 依赖关系是单向的，需要根据relationType判断方向
   - 延迟天数（delay）只对precedes/follows类型有意义

4. **缓存策略**：
   - 甘特图数据可以缓存，但需要在任务更新时刷新
   - 建议使用WebSocket实时更新任务状态

## 相关接口

- **获取任务列表**: `GET /api/issues`
- **获取任务详情**: `GET /api/issues/{id}`
- **获取任务树**: `GET /api/issues/tree`
- **获取任务统计**: `GET /api/issues/statistics`
- **更新任务**: `PUT /api/issues/{id}`
