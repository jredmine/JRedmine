# 论坛模块功能与接口分析

基于现有数据库（`boards`、`messages`）、权限枚举及附件/评论多态设计，整理论坛模块的功能与 REST 接口方案。

---

## 一、现有基础

| 项目 | 状态 | 说明 |
|------|------|------|
| 表结构 | ✅ | `boards`（板块）、`messages`（主题/回复） |
| 权限 | ✅ | view_messages、add_messages、edit_messages、delete_messages、manage_boards |
| 项目模块 | ✅ | ProjectModule.BOARDS，项目可启用/禁用论坛模块 |
| 评论 | ✅ | `comments` 表为多态（commented_type + commented_id），可关联 Message |
| 附件 | ✅ | 附件支持 containerType=Message、containerId=消息ID（需在附件模块约定或扩展） |
| 实体/Mapper | ❌ | 需新增 Board、Message 实体及对应 Mapper |

**说明**：论坛按项目组织，一个项目下可有多个**板块（Board）**；每个板块下为**消息（Message）**。消息通过 `parent_id` 区分：`parent_id` 为空表示**主题帖**，非空表示**回复**（指向主题 id）。主题有 subject、content、locked、sticky 等字段；回复无 subject（或沿用主题）。评论可挂到消息上（comments 表 commented_type='Message'）。

---

## 二、表结构摘要

### 1. boards（论坛板块）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| project_id | int | 所属项目 |
| name | varchar(255) | 板块名称 |
| description | varchar(255) | 板块描述 |
| position | int | 排序 |
| topics_count | int | 主题数（可冗余维护） |
| messages_count | int | 消息总数（可冗余维护） |
| last_message_id | int | 最后一条消息 ID（可冗余维护） |
| parent_id | int | 父板块 ID（可选，用于板块层级） |

### 2. messages（消息/主题与回复）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| board_id | int | 所属板块 |
| parent_id | int | 空=主题帖，非空=回复（指向主题 id） |
| subject | varchar(255) | 主题（主题帖必填；回复可空或沿用主题） |
| content | text | 正文 |
| author_id | int | 作者 |
| replies_count | int | 回复数（主题帖时有效，可冗余维护） |
| last_reply_id | int | 最后回复 id（可冗余维护） |
| created_on | datetime | 创建时间 |
| updated_on | datetime | 更新时间 |
| locked | tinyint(1) | 是否锁定（锁定后不可回复） |
| sticky | int | 是否置顶（置顶主题排在前面） |

**约定**：

- **主题帖**：`parent_id IS NULL`，`subject` 必填，`content` 可选。
- **回复**：`parent_id = 主题 id`，`subject` 可为空或与主题一致，`content` 必填。`locked`、`sticky` 仅对主题有效。

### 3. comments（评论，多态）

- `commented_type = 'Message'`、`commented_id = messages.id` 表示对某条消息的评论（若产品需要“消息下的评论”可复用此表；否则可仅用消息的回复树）。

## 三、功能概览

### 1. 板块（Board）

- **列表**：按项目查询所有板块，按 position、name 排序；可选返回每个板块的 topics_count、messages_count、last_message 摘要。
- **创建**：项目下新增板块（name 必填，description、position 可选）。权限：manage_boards。
- **详情**：按 boardId 返回板块信息及统计。
- **更新**：更新 name、description、position。权限：manage_boards。
- **删除**：删除板块；若板块下仍有消息，可禁止删除或级联删除（建议：禁止删除非空板块，或明确级联策略并同步更新 project 统计）。

### 2. 消息（Message）— 主题与回复

- **主题列表**：按板块分页查询**主题**（parent_id IS NULL），支持排序（如置顶优先、最后更新优先）、可选按关键词搜 subject/content。
- **创建主题**：在板块下新增一条 parent_id 为空的 message（subject 必填，content 可选）。权限：add_messages。
- **创建回复**：在主题下新增一条 parent_id=主题id 的 message（content 必填；subject 可选）。若主题已锁定则禁止回复。权限：add_messages。
- **主题详情**：按 messageId 返回主题及回复列表（可分页）；可选带评论数或评论列表（comments 表）。
- **更新消息**：更新 subject、content（主题）或 content（回复）；可更新主题的 locked、sticky。权限：本人 edit_messages 或 manage_boards。
- **删除消息**：删除主题时可选级联删除其下回复（及关联评论/附件）；删除回复则仅删该条。权限：本人 delete_messages 或 manage_boards。

### 3. 权限与可见性

- **项目模块**：仅当项目启用 `boards` 模块时，论坛接口对该项目可用。
- **权限**：
  - 板块/消息列表、主题与回复详情：view_messages。
  - 发主题、发回复：add_messages。
  - 编辑自己的消息、设置锁定/置顶：edit_messages（或 manage_boards 可编辑任意）。
  - 删除自己的消息：delete_messages（或 manage_boards 可删任意）。
  - 板块的增删改：manage_boards。

### 4. 与附件、评论的配合

- **附件**：若附件支持 containerType=Message、containerId=消息ID，则发帖/回复后上传附件走现有附件上传接口；消息详情可返回附件列表或附件数（与文档模块类似）。
- **评论**：若使用 `comments` 表，则 commented_type='Message'、commented_id=消息 id；消息详情可返回评论列表或评论数。

## 四、建议的 REST 接口清单

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/projects/{projectId}/boards` | 板块列表（含统计） | view_messages |
| POST | `/api/projects/{projectId}/boards` | 创建板块 | manage_boards |
| GET | `/api/projects/{projectId}/boards/{boardId}` | 板块详情（含统计） | view_messages |
| PUT | `/api/projects/{projectId}/boards/{boardId}` | 更新板块 | manage_boards |
| DELETE | `/api/projects/{projectId}/boards/{boardId}` | 删除板块（空板块或按策略级联） | manage_boards |
| GET | `/api/projects/{projectId}/boards/{boardId}/topics` | 主题分页列表（sticky 优先、updated_on 倒序；keyword 可选） | view_messages |
| POST | `/api/projects/{projectId}/boards/{boardId}/topics` | 发主题 | add_messages |
| GET | `/api/projects/{projectId}/boards/{boardId}/topics/{messageId}` | 主题详情（含回复列表，可分页） | view_messages |
| POST | `/api/projects/{projectId}/boards/{boardId}/topics/{messageId}/replies` | 回复主题 | add_messages |
| PUT | `/api/projects/{projectId}/boards/{boardId}/messages/{messageId}` | 更新消息（主题可更新 locked、sticky） | edit_messages / manage_boards |
| DELETE | `/api/projects/{projectId}/boards/{boardId}/messages/{messageId}` | 删除消息（主题可级联删回复） | delete_messages / manage_boards |

**附件**：上传/下载/删除消息附件使用现有附件接口，参数 `containerType=Message`、`containerId=消息ID`（若已支持）；否则需在附件模块扩展容器类型。

## 五、数据模型与 DTO 建议

### 1. 实体

- **Board**：id, projectId, name, description, position, topicsCount, messagesCount, lastMessageId, parentId。
- **Message**：id, boardId, parentId, subject, content, authorId, repliesCount, lastReplyId, createdOn, updatedOn, locked, sticky。

### 2. 请求 DTO

- **BoardCreateRequestDTO**：name（必填）, description, position。
- **BoardUpdateRequestDTO**：name, description, position（均可选）。
- **MessageCreateRequestDTO**（发主题）：subject（必填）, content。
- **ReplyCreateRequestDTO**（发回复）：content（必填）。
- **MessageUpdateRequestDTO**：subject, content, locked, sticky（均可选；回复一般只更新 content）。

### 3. 响应 DTO

- **BoardListItemResponseDTO**：id, projectId, name, description, position, topicsCount, messagesCount, lastMessageId；可选 lastMessageSubject、lastMessageUpdatedOn。
- **BoardDetailResponseDTO**：在列表项基础上增加父板块信息等（若需要）。
- **MessageTopicListItemResponseDTO**：id, boardId, subject, authorId, authorName, repliesCount, createdOn, updatedOn, locked, sticky。
- **MessageDetailResponseDTO**：主题或单条消息详情（id, boardId, parentId, subject, content, authorId, authorName, createdOn, updatedOn, locked, sticky）；主题详情可带 replies 分页列表。
- **MessageReplyListItemResponseDTO**：id, parentId, content, authorId, authorName, createdOn, updatedOn。

### 4. 统计维护策略

- **topics_count / messages_count / last_message_id**：在创建/删除消息时更新对应 board；主题的 replies_count、last_reply_id 在创建/删除回复时更新主题记录。
- 或采用“按需统计”减少写复杂度，列表/详情时实时 count 与 last 查询（适合数据量不大时）。

## 六、开发顺序建议

1. **Phase 1**：Board、Message 实体与 Mapper；项目启用 boards 模块校验；板块 CRUD；主题列表、发主题、主题详情、发回复；消息更新与删除（含主题级联删回复）；权限与项目模块校验。
2. **Phase 2**：主题列表支持置顶优先、关键词筛选；主题详情带回复分页；列表/详情返回作者名、最后更新信息；可选 boards 的 topics_count/messages_count 维护或按需统计。
3. **Phase 3**：消息附件（若附件支持 Message 容器）；消息评论（comments 表 Message 多态）；板块层级（parent_id）展示与筛选。

---

## 七、小结

| 类别 | 内容 |
|------|------|
| 表 | boards（板块）、messages（主题/回复） |
| 权限 | view_messages、add_messages、edit_messages、delete_messages、manage_boards |
| 核心接口 | 板块 CRUD；主题列表、发主题、主题详情、发回复；消息更新、删除 |
| 附件/评论 | 沿用或扩展现有附件 API（Message 容器）；评论用 comments 多态 |
| 与文档/Wiki 差异 | 论坛为“板块 + 主题/回复”树形结构，有锁定、置顶；无版本历史 |

按上述清单可实现与现有项目、权限、附件体系一致的论坛模块；实现时需补 Board、Message 实体与 Mapper，以及 BoardService、MessageService（或 BoardMessageService）、BoardController。
