# 任务关联关系查询功能实现说明

## 实现内容

### 1. 新增接口

#### GET /api/issues/{id}/relations
获取任务的所有关联关系列表

**请求示例**:
```bash
GET /api/issues/1/relations
Authorization: Bearer {token}
```

**响应示例**:
```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "id": 1,
      "issueFromId": 1,
      "issueFromSubject": "用户登录功能",
      "issueToId": 2,
      "issueToSubject": "用户注册功能",
      "relationType": "precedes",
      "delay": 0
    },
    {
      "id": 2,
      "issueFromId": 3,
      "issueFromSubject": "数据库设计",
      "issueToId": 1,
      "issueToSubject": "用户登录功能",
      "relationType": "blocks",
      "delay": null
    }
  ]
}
```

**特点**:
- ✅ 返回该任务作为源任务和目标任务的所有关联
- ✅ 包含关联任务的标题，方便展示
- ✅ 支持所有关联类型
- ✅ 需要 `view_issues` 权限

### 示例3：前端展示关联关系

```javascript
// 获取任务详情
fetch('/api/issues/1', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
})
.then(res => res.json())
.then(data => {
  const issue = data.data;
  const relations = issue.relations || [];
  
  // 按关联类型分组
  const grouped = {
    blocks: [],
    blocked: [],
    precedes: [],
    follows: [],
    duplicates: [],
    relates: [],
    copiedFrom: [],
    copiedTo: []
  };
  
  relations.forEach(rel => {
    // 判断当前任务是源还是目标
    if (rel.issueFromId === issue.id) {
      // 当前任务是源
      switch(rel.relationType) {
        case 'blocks':
          grouped.blocks.push({
            id: rel.issueToId,
            subject: rel.issueToSubject
          });
          break;
        case 'precedes':
          grouped.precedes.push({
            id: rel.issueToId,
            subject: rel.issueToSubject,
            delay: rel.delay
          });
          break;
        // ... 其他类型
      }
    } else {
      // 当前任务是目标
      switch(rel.relationType) {
        case 'blocks':
          grouped.blocked.push({
            id: rel.issueFromId,
            subject: rel.issueFromSubject
          });
          break;
        case 'precedes':
          grouped.follows.push({
            id: rel.issueFromId,
            subject: rel.issueFromSubject,
            delay: rel.delay
          });
          break;
        // ... 其他类型
      }
    }
  });
  
  // 展示关联关系
  if (grouped.blocked.length > 0) {
    console.log('⚠️ 该任务被以下任务阻塞:');
    grouped.blocked.forEach(task => {
      console.log(`  - #${task.id} ${task.subject}`);
    });
  }
  
  if (grouped.precedes.length > 0) {
    console.log('⏭️ 该任务的后续任务:');
    grouped.precedes.forEach(task => {
      console.log(`  - #${task.id} ${task.subject} (延迟: ${task.delay}天)`);
    });
  }
});
```

## 关联类型说明

| 关联类型 | 说明 | 源任务 | 目标任务 | 示例 |
|---------|------|--------|---------|------|
| relates | 相关 | A | B | A 与 B 相关 |
| duplicates | 重复 | A | B | A 是 B 的重复 |
| duplicated | 被重复 | A | B | A 被 B 重复 |
| blocks | 阻塞 | A | B | A 阻塞了 B |
| blocked | 被阻塞 | A | B | A 被 B 阻塞 |
| precedes | 前置 | A | B | A 在 B 之前完成 |
| follows | 后置 | A | B | A 在 B 之后开始 |
| copied_to | 复制到 | A | B | A 被复制为 B |
| copied_from | 从...复制 | A | B | A 从 B 复制而来 |

### 关联关系的方向性

需要注意，关联关系是有方向的：

- **blocks**: A blocks B 表示 "A阻塞了B"
  - 从A的角度：A阻塞了B（blocks）
  - 从B的角度：B被A阻塞（blocked by A）

- **precedes**: A precedes B 表示 "A在B之前"
  - 从A的角度：A的后续任务是B（precedes B）
  - 从B的角度：B的前置任务是A（follows A）

### 3. 甘特图中的关联线

```javascript
// 在甘特图中绘制关联线
function drawRelationLines(tasks, relations) {
  relations.forEach(rel => {
    const fromTask = tasks.find(t => t.id === rel.issueFromId);
    const toTask = tasks.find(t => t.id === rel.issueToId);
    
    if (fromTask && toTask) {
      const color = getRelationColor(rel.relationType);
      const style = getRelationStyle(rel.relationType);
      
      // 绘制连接线
      drawLine(fromTask, toTask, color, style);
      
      // 添加延迟标签
      if (rel.delay && rel.delay > 0) {
        addDelayLabel(fromTask, toTask, rel.delay);
      }
    }
  });
}

function getRelationColor(type) {
  switch(type) {
    case 'blocks': return '#ff0000';   // 红色 - 阻塞
    case 'precedes': return '#0000ff'; // 蓝色 - 前置
    case 'relates': return '#999999';  // 灰色 - 相关
    default: return '#000000';
  }
}

function getRelationStyle(type) {
  switch(type) {
    case 'blocks': return 'solid';    // 实线 - 阻塞
    case 'precedes': return 'dashed'; // 虚线 - 前置
    case 'relates': return 'dotted';  // 点线 - 相关
    default: return 'solid';
  }
}
```

## 注意事项

1. **权限控制**：
   - 需要 `view_issues` 权限才能查看关联关系
   - 私有任务的关联仅项目成员可见

2. **性能考虑**：
   - 每次查询任务详情都会查询关联关系
   - 如果任务有很多关联，可能影响性能
   - 建议在列表接口中不返回关联关系

3. **数据一致性**：
   - 关联关系是单向的（在数据库中）
   - 前端需要根据 `issueFromId` 和 `issueToId` 判断方向
   - 对称关系（blocks/blocked, precedes/follows等）需要前端自行处理展示

4. **未来改进**：
   - 考虑实现对称关系自动创建
   - 添加关联关系的业务逻辑（如阻塞检查）
   - 优化性能（批量查询、缓存等）
