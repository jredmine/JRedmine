# 工时记录批量导入功能实现说明

## 概述

工时记录批量导入功能允许用户通过**Excel**或**CSV**文件一次性导入大量工时记录，大幅提高数据录入效率，特别适合历史数据迁移、周期性工时录入等场景。

**开发日期**：2026-01-14  
**开发耗时**：约2小时

## 功能特性

### 支持的导入格式

| 格式 | 特点 | 适用场景 |
|------|------|---------|
| **Excel (.xlsx)** | 可视化编辑、格式友好 | 手工填写、数据整理 |
| **CSV (.csv)** | 轻量级、程序生成 | 系统导出、批量生成 |

### 核心功能

1. **模板下载**
   - 提供Excel和CSV两种格式的标准模板
   - 包含示例数据和字段说明
   - 表头清晰标注必填项

2. **智能解析**
   - 支持ID和名称两种方式
   - 自动跳过空行
   - 友好的错误提示

3. **批量处理**
   - 支持一次导入多条记录
   - 部分成功机制
   - 详细的失败信息反馈

4. **数据验证**
   - 必填项检查
   - 格式验证
   - 关联数据存在性验证

## 导入字段说明

| 字段 | 是否必填 | 说明 | 示例 |
|------|----------|------|------|
| **项目ID或标识*** | 必填 | 可以填项目ID或项目标识符 | `1` 或 `demo-project` |
| **任务ID** | 可选 | 关联的任务ID | `100` |
| **用户ID或登录名*** | 必填 | 可以填用户ID或登录名 | `1` 或 `admin` |
| **活动类型ID或名称*** | 必填 | 可以填活动类型ID或名称 | `9` 或 `开发` |
| **工作日期(yyyy-MM-dd)*** | 必填 | 格式必须为 yyyy-MM-dd | `2026-01-14` |
| **工时(小时)*** | 必填 | 小时数，可以是小数 | `8.0` 或 `2.5` |
| **备注** | 可选 | 工作内容说明 | `完成用户登录功能` |

### Excel导入实现

#### 文件解析

```java
public TimeEntryBatchImportResponseDTO importFromExcel(MultipartFile file) {
    try (Workbook workbook = new XSSFWorkbook(file.getInputStream())) {
        Sheet sheet = workbook.getSheetAt(0);
        
        // 跳过表头，从第二行开始
        for (Row row : sheet) {
            if (rowNum == 1) continue;  // 跳过表头
            if (isEmptyRow(row)) continue;  // 跳过空行
            
            // 解析行数据
            TimeEntryBatchImportRequestDTO record = parseExcelRow(row, rowNum);
            records.add(record);
        }
    }
    
    return processImport(records, failures);
}
```

private String getCellStringValue(Cell cell) {
    switch (cell.getCellType()) {
        case STRING:
            return cell.getStringCellValue();
        case NUMERIC:
            if (DateUtil.isCellDateFormatted(cell)) {
                return DATE_FORMATTER.format(cell.getLocalDateTimeCellValue());
            }
            // 数字类型处理
            double numValue = cell.getNumericCellValue();
            if (numValue == (long) numValue) {
                return String.valueOf((long) numValue);  // 整数
            }
            return String.valueOf(numValue);  // 小数
        // ... 其他类型
    }
}

### CSV导入实现

```java
public TimeEntryBatchImportResponseDTO importFromCSV(MultipartFile file) {
    try (InputStreamReader reader = new InputStreamReader(file.getInputStream(), UTF_8);
         CSVParser csvParser = new CSVParser(reader, 
                 CSVFormat.DEFAULT.withFirstRecordAsHeader())) {
        
        for (CSVRecord record : csvParser) {
            TimeEntryBatchImportRequestDTO dto = parseCSVRecord(record, rowNum);
            records.add(dto);
        }
    }
    
    return processImport(records, failures);
}
```

private Long parseProjectId(String projectStr) {
    // 尝试作为ID解析
    try {
        return Long.parseLong(projectStr.trim());
    } catch (NumberFormatException e) {
        // 作为项目标识查询
        Project project = projectMapper.selectOne(
            new LambdaQueryWrapper<Project>()
                .eq(Project::getIdentifier, projectStr.trim())
        );
        
        if (project == null) {
            throw new BusinessException("项目不存在: " + projectStr);
        }
        
        return project.getId();
    }
}

private Long parseActivityId(String activityStr) {
    // 尝试作为ID解析
    try {
        return Long.parseLong(activityStr.trim());
    } catch (NumberFormatException e) {
        // 作为活动类型名称查询
        Enumeration enumeration = enumerationMapper.selectOne(
            new LambdaQueryWrapper<Enumeration>()

                .eq(Enumeration::getType, "TimeEntryActivity")

        if (enumeration == null) {
            throw new BusinessException("活动类型不存在: " + activityStr);
        }
        
        return enumeration.getId();
    }
}

### 批量处理机制

```java
private TimeEntryBatchImportResponseDTO processImport(
        List<TimeEntryBatchImportRequestDTO> records,
        List<FailureDetail> failures) {
    
    int successCount = 0;
    
    for (int i = 0; i < records.size(); i++) {
        TimeEntryBatchImportRequestDTO record = records.get(i);
        
        try {
            // 转换为创建请求DTO
            TimeEntryCreateRequestDTO createDTO = convertToCreateDTO(record);
            
            // 调用创建服务
            timeEntryService.createTimeEntry(createDTO);
            successCount++;
            
        } catch (Exception e) {
            // 记录失败信息
            failures.add(FailureDetail.builder()
                    .rowNumber(i + 2)  // 行号（加2因为跳过了表头）
                    .reason(e.getMessage())
                    .rawData(record.toString())
                    .build());
        }
    }
    
    return TimeEntryBatchImportResponseDTO.builder()
            .totalCount(records.size() + failures.size())
            .successCount(successCount)
            .failureCount(failures.size())
            .failures(failures)
            .build();
}
```

### 模板生成

#### Excel模板

```java
public byte[] generateExcelTemplate() {
    try (Workbook workbook = new XSSFWorkbook();
         ByteArrayOutputStream out = new ByteArrayOutputStream()) {
        
        Sheet sheet = workbook.createSheet("工时记录导入模板");
        
        // 表头（灰色背景、加粗）
        Row headerRow = sheet.createRow(0);
        String[] headers = {
            "项目ID或标识*", "任务ID", "用户ID或登录名*", "活动类型ID或名称*",
            "工作日期(yyyy-MM-dd)*", "工时(小时)*", "备注"
        };
        
        for (int i = 0; i < headers.length; i++) {
            Cell cell = headerRow.createCell(i);
            cell.setCellValue(headers[i]);
            cell.setCellStyle(headerStyle);
        }
        
        // 示例数据（灰色字体）
        Row exampleRow = sheet.createRow(1);
        String[] examples = {
            "1", "100", "admin", "开发", "2026-01-14", "8.0", "完成用户登录功能"
        };
        
        for (int i = 0; i < examples.length; i++) {
            Cell cell = exampleRow.createCell(i);
            cell.setCellValue(examples[i]);
            cell.setCellStyle(exampleStyle);
        }
        
        // 说明行
        Row noteRow = sheet.createRow(3);
        Cell noteCell = noteRow.createCell(0);
        noteCell.setCellValue("说明：带*为必填项。项目和用户可以填ID或标识/登录名。");
        
        // 自动调整列宽
        for (int i = 0; i < headers.length; i++) {
            sheet.autoSizeColumn(i);
        }
        
        workbook.write(out);
        return out.toByteArray();
    }
}
```

#### CSV模板

```java
public byte[] generateCSVTemplate() {
    try (ByteArrayOutputStream out = new ByteArrayOutputStream();
         Writer writer = new OutputStreamWriter(out, UTF_8);
         CSVPrinter csvPrinter = new CSVPrinter(writer, CSVFormat.DEFAULT)) {
        
        // 添加BOM（Excel兼容）
        out.write(0xEF);
        out.write(0xBB);
        out.write(0xBF);
        
        // 表头
        csvPrinter.printRecord(
            "项目ID或标识*", "任务ID", "用户ID或登录名*", "活动类型ID或名称*",
            "工作日期(yyyy-MM-dd)*", "工时(小时)*", "备注"
        );
        
        // 示例数据
        csvPrinter.printRecord(
            "1", "100", "admin", "开发", "2026-01-14", "8.0", "完成用户登录功能"
        );
        
        csvPrinter.printRecord(
            "demo-project", "101", "developer", "测试", "2026-01-15", "4.0", "编写单元测试"
        );
        
        csvPrinter.flush();
        return out.toByteArray();
    }
}

  -F "file=@/tmp/auto_import.csv")

failure_count=$(echo "$response" | jq -r '.data.failureCount')

# 4. 发送通知（如果有失败）

    send_alert_email "工时导入有 $failure_count 条失败"
fi
```

## 错误处理

### 常见错误及解决方案

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| 项目不存在: xxx | 项目ID或标识不存在 | 检查项目是否存在，使用正确的ID或标识 |
| 用户不存在: xxx | 用户ID或登录名不存在 | 检查用户是否存在，使用正确的ID或登录名 |
| 活动类型不存在: xxx | 活动类型ID或名称不存在 | 检查活动类型是否存在，使用正确的ID或名称 |
| 工作日期不能为空 | 日期字段为空 | 填写日期，格式为 yyyy-MM-dd |
| 无效的日期格式: xxx | 日期格式不正确 | 使用正确的日期格式 yyyy-MM-dd |
| 工时不能为空 | 工时字段为空 | 填写工时，必须大于0 |
| 无效的工时值: xxx | 工时格式不正确 | 填写数字，可以是小数如 8.0 或 2.5 |
| 数据格式错误 | 单元格数据类型不正确 | 检查每个字段的数据类型 |

### 响应状态码

| 状态码 | 说明 | 场景 |
|-------|------|------|
| 200 | 成功 | 全部导入成功或部分成功 |
| 400 | 请求错误 | 文件格式错误、全部导入失败 |
| 401 | 未授权 | Token无效或过期 |
| 403 | 权限不足 | 没有log_time权限 |
| 500 | 服务器错误 | 服务器内部错误 |

## 最佳实践

### 1. 数据准备

**使用ID vs 使用名称**：
- ✅ **推荐**：优先使用ID（更快、更准确）
- ⚠️ **可选**：使用名称（更直观，但需确保唯一性）

**日期格式**：
- ✅ 使用 `2026-01-14` 格式
- ❌ 避免使用 `2026/01/14` 或 `01-14-2026`

**工时格式**：
- ✅ 使用数字：`8.0`、`2.5`
- ❌ 避免使用文本：`8小时`、`half day`

### 2. 文件大小

| 文件大小 | 记录数 | 处理时间 | 建议 |
|---------|--------|---------|------|
| < 100KB | < 1000条 | 秒级 | 直接导入 |
| 100KB - 1MB | 1000 - 10000条 | 分钟级 | 分批导入 |
| > 1MB | > 10000条 | 较长 | 分多个文件 |

### 3. 错误处理策略

**部分成功处理**：
1. 查看失败记录的详细信息
2. 修正失败记录的数据
3. 仅重新导入失败的记录

**全部失败处理**：
1. 检查文件格式是否正确
2. 检查必填字段是否完整
3. 检查数据类型是否正确
4. 修正后重新导入

### 4. 性能优化

**大批量导入**：
- 分批导入，每批不超过1000条
- 在非高峰期导入
- 导入前做好数据验证

**并发控制**：
- 避免同时导入多个文件
- 使用队列机制处理大批量导入
