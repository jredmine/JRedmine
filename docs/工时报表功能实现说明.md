# 工时报表功能实现说明

## 概述

工时报表功能是工时记录模块的高级分析工具,提供三种专业的报表视图,帮助管理者从不同角度深入分析工时数据,为项目管理和资源调配提供数据支持。

**开发日期**:2026年1月13日  
**开发耗时**:约1.5小时

## 功能特性

### 1. 项目工时报表

**用途**:全面分析单个项目的工时使用情况

**包含内容**:
- **基础统计**:总工时、记录数、参与人数、平均工时
- **时间范围**:最早和最晚记录日期
- **成员分布**:各成员的工时、记录数、占比排名
- **活动分布**:各活动类型的工时、记录数、占比排名

**应用场景**:
- 项目成本核算
- 团队资源评估
- 工作量分析
- 项目进度跟踪

### 2. 用户工时报表

**用途**:全面分析单个用户的工时使用情况

**包含内容**:
- **基础统计**:总工时、记录数、参与项目数、平均工时
- **时间范围**:最早和最晚记录日期
- **项目分布**:各项目的工时、记录数、占比排名
- **活动分布**:各活动类型的工时、记录数、占比排名

**应用场景**:
- 个人绩效评估
- 工作负荷分析
- 工作内容分析
- 多项目协调

### 3. 时间段工时报表

**用途**:分析工时的时间变化趋势

**包含内容**:
- **基础统计**:总工时、记录数、平均每日工时
- **极值统计**:最高和最低单日工时
- **趋势数据**:各时间段的工时、记录数、参与人数
- **时间粒度**:支持按日、周、月三种粒度

**应用场景**:
- 工时趋势分析
- 工作高峰识别
- 资源规划
- 进度监控

    // 5. 按项目分组统计并计算占比

#### 3. 时间段报表生成算法

```java
public TimeEntryPeriodReportDTO generatePeriodReport(request) {
    // 1. 构建查询条件(可选项目/用户 + 日期范围)
    // 2. 查询所有工时记录
    // 3. 确定时间段类型(day/week/month)
    // 4. 计算基础统计(总工时、记录数等)
    // 5. 按时间段分组统计
    // 6. 计算每个时间段的参与人数
    // 7. 计算平均/最大/最小单日工时
    // 8. 按时间顺序排序
    // 9. 返回报表数据
}
```

**关键点**:
- 支持三种时间粒度(日、周、月)
- 使用`getPeriodKey()`生成统一的时间段标识
- 按时间顺序排序,便于趋势分析

#### 4. 时间段键生成算法

```java
private String getPeriodKey(LocalDate date, String periodType) {
    switch (periodType) {
        case "week":
            // 生成格式: 2026-W02
            return String.format("%d-W%02d", year, weekNumber);
        case "month":
            // 生成格式: 2026-01
            return String.format("%d-%02d", year, month);
        case "day":
        default:
            // 生成格式: 2026-01-13
            return date.toString();
    }
}
```

### API接口

#### 1. 项目工时报表

```
GET /api/time-entries/reports/project
```

**权限**:`view_time_entries`

**参数**:
- `projectId` (必填):项目ID
- `startDate` (可选):开始日期
- `endDate` (可选):结束日期
- `year` (可选):年份
- `month` (可选):月份

**示例**:
```bash
# 查询项目1的全部工时报表
GET /api/time-entries/reports/project?projectId=1

# 查询项目1在2026年1月的工时报表
GET /api/time-entries/reports/project?projectId=1&year=2026&month=1

# 查询项目1指定日期范围的工时报表
GET /api/time-entries/reports/project?projectId=1&startDate=2026-01-01&endDate=2026-01-31
```

#### 2. 用户工时报表

```
GET /api/time-entries/reports/user
```

**权限**:`view_time_entries`

**参数**:
- `userId` (必填):用户ID
- `startDate` (可选):开始日期
- `endDate` (可选):结束日期
- `year` (可选):年份
- `month` (可选):月份

**示例**:
```bash
# 查询用户1的全部工时报表
GET /api/time-entries/reports/user?userId=1

# 查询用户1在2026年1月的工时报表
GET /api/time-entries/reports/user?userId=1&year=2026&month=1
```

#### 3. 时间段工时报表

```
GET /api/time-entries/reports/period
```

**权限**:`view_time_entries`

**参数**:
- `periodType` (可选):时间段类型(day/week/month),默认day
- `projectId` (可选):项目ID
- `userId` (可选):用户ID
- `startDate` (可选):开始日期
- `endDate` (可选):结束日期
- `year` (可选):年份
- `month` (可选):月份

**示例**:
```bash
# 按日查询2026年1月的工时趋势
GET /api/time-entries/reports/period?periodType=day&year=2026&month=1

# 按周查询2026年1月的工时趋势
GET /api/time-entries/reports/period?periodType=week&year=2026&month=1

# 按月查询2026年全年的工时趋势
GET /api/time-entries/reports/period?periodType=month&year=2026

# 查询项目1在2026年1月的每日工时趋势
GET /api/time-entries/reports/period?projectId=1&periodType=day&year=2026&month=1
```

## 性能优化

### 1. 数据查询优化
- 一次性查询所有数据,避免多次数据库交互
- 使用HashMap进行内存分组,提高效率
- 利用Stream API进行集合操作

### 2. 计算优化
- 在内存中完成所有统计计算
- 避免重复查询数据库
- 使用缓存存储常用的实体信息

### 3. 数据精度
- 所有百分比和工时保留2位小数
- 使用`round()`方法统一处理精度

### 4. 空数据处理
- 提供专门的空报表构建方法
- 避免空指针异常
- 返回有意义的默认值
