# 用户管理模块接口清单

> 本文档列出 JRedmine 用户管理模块需要实现的所有接口，按照优先级分类。

## ⚠️ 重要说明：权限控制

### 系统用户权限体系

**用户表（users）**：
- 只有 `admin` 字段，用于区分是否是**系统管理员**
- 系统管理员（`admin = true`）拥有所有权限
- 普通用户（`admin = false`）只能操作自己的数据

### 权限控制规则

**仅管理员可操作的接口**：
- ✅ 创建用户（`POST /api/users`）
- ✅ 删除用户（`DELETE /api/users/{id}`）
- ✅ 更新用户信息（`PUT /api/users/{id}`）- 包括管理员权限、状态等敏感字段
- ✅ 更新用户状态（`PUT /api/users/{id}/status`）
- ✅ 查询用户列表（`GET /api/users`）

**管理员或用户自己可操作的接口**：
- ✅ 查询用户详情（`GET /api/users/{id}`）- 用户可以查询自己的，管理员可以查询所有
- ✅ 更新用户偏好设置（`PUT /api/users/{id}/preferences`）- 用户可以更新自己的，管理员可以更新所有

**所有用户可操作的接口**：
- ✅ 获取当前用户信息（`GET /api/users/me`）
- ✅ 获取当前用户偏好设置（`GET /api/users/me/preferences`）
- ✅ 更新当前用户偏好设置（`PUT /api/users/me/preferences`）
- ✅ 变更密码（`POST /api/auth/change-password`）- 只能修改自己的密码

## 🔴 核心接口（P0 - 必须实现）

### 1. 用户注册 ✅

**接口**: `POST /api/users/register`

**状态**: ✅ 已实现

**功能**: 注册新用户账号

**请求体**:
```json
{
  "login": "username",
  "password": "password123",
  "confirmPassword": "password123",
  "firstname": "张",
  "lastname": "三",
  "email": "zhangsan@example.com",
  "hideEmailFlag": false
}
```

**响应**: `ApiResponse<UserRegisterResponseDTO>`

---

### 2. 用户登录 ⚠️

**接口**: `POST /api/users/login`

**状态**: ❌ 未实现

**功能**: 用户登录认证，返回 JWT Token

**请求体**:
```json
{
  "login": "username",
  "password": "password123"
}
```

**响应**:
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzUxMiJ9...",
    "tokenType": "Bearer",
    "expiresIn": 86400,
    "user": {
      "id": 1,
      "login": "username",
      "firstname": "张",
      "lastname": "三",
      "email": "zhangsan@example.com",
      "admin": false,
      "status": 1
    }
  }
}
```

**业务逻辑**:
- 验证用户名和密码
- 更新 `lastLoginOn` 字段
- 生成 JWT Token
- 记录登录日志

---

### 3. 用户信息查询 ✅

**接口**: `GET /api/users/{id}`

**状态**: ✅ 已实现

**功能**: 根据用户ID查询用户详细信息

**路径参数**: `id` - 用户ID

**响应**: `ApiResponse<UserDetailResponseDTO>`

**权限**: ⚠️ **用户可以查询自己的信息，管理员可以查询所有用户信息**

---

### 4. 用户列表查询 ✅

**接口**: `GET /api/users`

**状态**: ✅ 已实现

**功能**: 分页查询用户列表，支持按登录名模糊查询

**查询参数**:
- `current` (Integer, 默认1) - 当前页码
- `size` (Integer, 默认10) - 每页大小
- `login` (String, 可选) - 登录名（模糊查询）

**响应**: `ApiResponse<PageResponse<UserListItemResponseDTO>>`

**权限**: ⚠️ **仅管理员可访问**

---

### 5. 用户信息更新 ✅

**接口**: `PUT /api/users/{id}`

**状态**: ✅ 已实现

**功能**: 更新用户信息（包括姓名、邮箱、管理员权限、用户状态等）

**路径参数**: `id` - 用户ID

**请求体**:
```json
{
  "firstname": "张",
  "lastname": "三",
  "email": "zhangsan@example.com",
  "admin": false,
  "status": 1,
  "language": "zh-CN",
  "mailNotification": "all"
}
```

**响应**: `ApiResponse<UserDetailResponseDTO>`

**权限**: ⚠️ **仅管理员可访问**（因为可以修改管理员权限、用户状态等敏感字段）

---

### 6. 用户密码修改 ⚠️

**接口**: `PUT /api/users/{id}/password`

**状态**: ❌ 未实现

**功能**: 修改用户密码

**路径参数**: `id` - 用户ID

**请求体**:
```json
{
  "oldPassword": "oldPassword123",
  "newPassword": "newPassword123",
  "confirmPassword": "newPassword123"
}
```

**响应**: `ApiResponse<Void>`

**权限**: 
- 用户可修改自己的密码
- 管理员可修改所有用户密码（无需旧密码）

**业务逻辑**:
- 验证旧密码（非管理员）
- 验证新密码强度
- 更新密码和 `passwdChangedOn` 字段
- 清除 `mustChangePasswd` 标志

---

### 7. 用户状态管理 ✅

**接口**: `PUT /api/users/{id}/status`

**状态**: ✅ 已实现

**功能**: 启用/禁用/锁定用户账号

**路径参数**: `id` - 用户ID

**请求体**:
```json
{
  "status": 1  // 1-启用, 2-锁定, 3-待激活
}
```

**响应**: `ApiResponse<UserDetailResponseDTO>`

**权限**: ⚠️ **仅管理员可访问**

**业务逻辑**:
- 验证状态值（1-启用, 2-禁用, 3-锁定）
- 更新用户状态
- 禁用时清除用户会话（如果使用Session）
